#[prefix_logs_with]
Expand description

Add a log prefix to the function.

This prefixes all the log lines with [<name>] (after the timestamp). It works by making a tracing‚Äôs span that is propagated to all the child calls and child tasks (futures) if they are spawned properly with the SpawnHandle (see TaskManager in sc-cli) or if the futures use .in_current_span() (see tracing-futures).

See Tokio‚Äôs tracing documentation and tracing-futures documentation for more details.

¬ßImplementation notes

If there are multiple spans with a log prefix, only the latest will be shown.
