Expand description

Pair which is able to verify proof of possession.

While you donโt need a keypair to verify a proof of possession (you only need a public key) we constrain on Pair to use the Public and Signature types associated to Pair. This is implemented in different trait (than Public Key) to provide default behavior.

Verify proof of possession.

The proof of possession verifier is supposed to to verify a signature with unique hash context that is produced solely for this reason. This proves that that the secret key is known to the prover.
