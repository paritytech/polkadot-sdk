19//! [`Peerset`] implementation for `litep2p`.
20//!
21//! [`Peerset`] is a separate but related component running alongside the notification protocol,
22//! responsible for maintaining connectivity to remote peers. [`Peerset`] has an imperfect view of
23//! the network as the notification protocol is behind an asynchronous channel. Based on this
24//! imperfect view, it tries to connect to remote peers and disconnect peers that should be
25//! disconnected from.
26//!
27//! [`Peerset`] knows of two types of peers:
28//!  - normal peers
29//!  - reserved peers
30//!
31//! Reserved peers are those which the [`Peerset`] should be connected at all times and it will make
32//! an effort to do so by constantly checking that there are no disconnected reserved peers (except
33//! banned) and if there are, it will open substreams to them.
34//!
35//! [`Peerset`] may also contain "slots", both inbound and outbound, which mark how many incoming
36//! and outgoing connections it should maintain at all times. Peers for the inbound slots are filled
37//! by remote peers opening inbound substreams towards the local node and peers for the outbound
38//! slots are filled by querying the `Peerstore` which contains all peers known to `sc-network`.
39//! Peers for outbound slots are selected in a decreasing order of reputation.

90/// Is the peer reserved?

92/// Regular peers count towards slot allocation.

94pub enum Reserved {
95	Yes,
96	No,
97}

99impl From<bool> for Reserved {
100	fn from(value: bool) -> Reserved {
101		match value {
102			true => Reserved::Yes,
103			false => Reserved::No,
104		}
105	}
106}

108impl From<Reserved> for bool {
109	fn from(value: Reserved) -> bool {
110		std::matches!(value, Reserved::Yes)
111	}

123impl Direction {
124	fn set_reserved(&mut self, new_reserved: Reserved) {
125		match self {
126			Direction::Inbound(ref mut reserved) | Direction::Outbound(ref mut reserved) =>

128		}
129	}

133	fn from(direction: Direction) -> traits::Direction {
134		match direction {

136			Direction::Outbound(_) => traits::Direction::Outbound,
137		}
138	}

220/// Peer state.
221///
222/// Peer can be in 6 different state:
223///  - disconnected
224///  - connected
225///  - connection is opening
226///  - connection is closing
227///  - connection is backed-off
228///  - connection is canceled
229///
230/// Opening and closing are separate states as litep2p guarantees to report when the substream is
231/// either fully open or fully closed and the slot allocation for opening a substream is tied to a
232/// state transition which moves the peer to [`PeerState::Opening`]. This is because it allows
233/// reserving a slot for peer to prevent infinite outbound substreams. If the substream is opened
234/// successfully, peer is moved to state [`PeerState::Connected`] but there is no modification to
235/// the slot count as an outbound slot was already allocated for the peer. If the substream fails to
236/// open, the event is reported by litep2p and [`Peerset::report_substream_open_failure()`] is
237/// called which will decrease the outbound slot count. Similarly for inbound streams, the slot is
238/// allocated in [`Peerset::report_inbound_substream()`] which will prevent `Peerset` from accepting
239/// infinite inbound substreams. If the inbound substream fails to open and since [`Peerset`] was
240/// notified of it, litep2p will report the open failure and the inbound slot count is once again
241/// decreased in [`Peerset::report_substream_open_failure()`]. If the substream is opened
242/// successfully, the slot count is not modified.
243///
244/// Since closing a substream is not instantaneous, there is a separate [`PeerState::Closing`]
245/// state which indicates that the substream is being closed but hasn't been closed by litep2p yet.
246/// This state is used to prevent invalid state transitions where, for example, [`Peerset`] would
247/// close a substream and then try to reopen it immediately.
248///
249/// Irrespective of which side closed the substream (local/remote), the substream is chilled for a
250/// small amount of time ([`DEFAULT_BACKOFF`]) and during this time no inbound or outbound
251/// substreams are accepted/established. Any request to open an outbound substream while the peer
252/// is backed-off is ignored. If the peer is a reserved peer, an outbound substream is not opened
253/// for them immediately but after the back-off has expired, `Peerset` will attempt to open a
254/// substream to the peer if it's still counted as a reserved peer.
255///
256/// Disconnections and open failures will contribute negatively to the peer score to prevent it from
257/// being selected for another outbound substream request soon after the failure/disconnection. The
258/// reputation decays towards zero over time and eventually the peer will be as likely to be
259/// selected for an outbound substream as any other freshly added peer.
260///
261/// [`Peerset`] must also be able to handle the case where an outbound substream was opened to peer
262/// and while it was opening, an inbound substream was received from that same peer. Since `litep2p`
263/// is the source of truth of the actual state of the connection, [`Peerset`] must compensate for
264/// this and if it happens that inbound substream is opened for a peer that was marked outbound, it
265/// will attempt to allocate an inbound slot for the peer. If it fails to do so, the inbound
266/// substream is rejected and the peer is marked as canceled.
267///
268/// Since substream is not opened immediately, a peer can be disconnected even if the substream was
269/// not yet open. This can happen, for example, when a peer has connected over the syncing protocol
270/// and it was added to, e.g., GRANDPA's reserved peers, an outbound substream was opened
271/// ([`PeerState::Opening`]) and then the peer disconnected. This state transition is handled by the
272/// [`Peerset`] with `PeerState::Canceled` which indicates that should the substream open
273/// successfully, it should be closed immediately and if the connection is opened successfully while
274/// the peer was marked as canceled, the substream will be closed without notifying the protocol
275/// about the substream.
276#[derive(Debug, PartialEq, Eq)]
277pub enum PeerState {
278	/// No active connection to peer.
279	Disconnected,
280
281	/// Substream to peer was recently closed and the peer is currently backed off.
282	///
283	/// Backoff only applies to outbound substreams. Inbound substream will not experience any sort
284	/// of "banning" even if the peer is backed off and an inbound substream for the peer is
285	/// received.
286	Backoff,
287
288	/// Connection to peer is pending.
289	Opening {
290		/// Direction of the connection.
291		direction: Direction,
292	},
293
294	// Connected to peer.
295	Connected {
296		/// Is the peer inbound or outbound.
297		direction: Direction,
298	},
299
300	/// Substream was opened and while it was opening (no response had been heard from litep2p),
301	/// the substream was canceled by either calling `disconnect_peer()` or by removing peer
302	/// from the reserved set.
303	///
304	/// After the opened substream is acknowledged by litep2p (open success/failure), the peer is
305	/// moved to [`PeerState::Backoff`] from which it will then be moved to
306	/// [`PeerState::Disconnected`].
307	Canceled {
308		/// Is the peer inbound or outbound.
309		direction: Direction,
310	},
311
312	/// Connection to peer is closing.
313	///
314	/// State implies that the substream was asked to be closed by the local node and litep2p is
315	/// closing the substream. No command modifying the connection state is accepted until the
316	/// state has been set to [`PeerState::Disconnected`].
317	Closing {
318		/// Is the peer inbound or outbound.
319		direction: Direction,
320	},
321}

323/// `Peerset` implementation.
324///
325/// `Peerset` allows other subsystems of the blockchain to modify the connection state
326/// of the notification protocol by adding and removing reserved peers.
327///
328/// `Peerset` is also responsible for maintaining the desired amount of peers the protocol is
329/// connected to by establishing outbound connections and accepting/rejecting inbound connections.
330#[derive(Debug)]
331pub struct Peerset {
332	/// Protocol name.
333	protocol: ProtocolName,
334
335	/// RX channel for receiving commands.
336	cmd_rx: TracingUnboundedReceiver<PeersetCommand>,
337
338	/// Maximum number of outbound peers.
339	max_out: usize,
340
341	/// Current number of outbound peers.
342	num_out: usize,
343
344	/// Maximum number of inbound peers.
345	max_in: usize,
346
347	/// Current number of inbound peers.
348	num_in: usize,
349
350	/// Only connect to/accept connections from reserved peers.
351	reserved_only: bool,
352
353	/// Current reserved peer set.
354	reserved_peers: HashSet<PeerId>,
355
356	/// Handle to `Peerstore`.
357	peerstore_handle: Arc<dyn PeerStoreProvider>,
358
359	/// Peers.
360	peers: HashMap<PeerId, PeerState>,
361
362	/// Counter connected peers.
363	connected_peers: Arc<AtomicUsize>,
364
365	/// Pending backoffs for peers who recently disconnected.
366	pending_backoffs: FuturesUnordered<BoxFuture<'static, (PeerId, Reputation)>>,
367
368	/// Next time when [`Peerset`] should perform slot allocation.
369	next_slot_allocation: Delay,
370}

372macro_rules! decrement_or_warn {
373    ($slot:expr, $protocol:expr, $peer:expr, $direction:expr) => {{
374		match $slot.checked_sub(1) {
375			Some(value) => {
376				$slot = value;
377			}
378			None => {
379				log::warn!(
380					target: LOG_TARGET,
381					"{}: state mismatch, {:?} is not counted as part of {:?} slots",
382					$protocol, $peer, $direction
383				);
384				debug_assert!(false);
385			}
386		}
387    }};
388}

405	/// Create new [`Peerset`].
406	pub fn new(
407		protocol: ProtocolName,
408		max_out: usize,
409		max_in: usize,
410		reserved_only: bool,
411		reserved_peers: HashSet<PeerId>,
412		connected_peers: Arc<AtomicUsize>,
413		peerstore_handle: Arc<dyn PeerStoreProvider>,
414	) -> (Self, TracingUnboundedSender<PeersetCommand>) {
415		let (cmd_tx, cmd_rx) = tracing_unbounded("mpsc-peerset-protocol", 100_000);
416		let peers = reserved_peers
417			.iter()
418			.map(|peer| (*peer, PeerState::Disconnected))
419			.collect::<HashMap<_, _>>();
420
421		// register protocol's command channel to `Peerstore` so it can issue disconnect commands
422		// if some connected peer gets banned.
423		peerstore_handle.register_protocol(Arc::new(PeersetHandle { tx: cmd_tx.clone() }));
424
425		log::debug!(
426			target: LOG_TARGET,
427			"{}: creating new peerset with max_outbound {} and max_inbound {} and reserved_only {}",
428			protocol,
429			max_out,
430			max_in,
431			reserved_only,
432		);
433
434		(
435			Self {
436				protocol,
437				max_out,
438				num_out: 0usize,
439				max_in,
440				num_in: 0usize,
441				reserved_peers,
442				cmd_rx,
443				peerstore_handle,
444				reserved_only,
445				peers,
446				connected_peers,
447				pending_backoffs: FuturesUnordered::new(),
448				next_slot_allocation: Delay::new(SLOT_ALLOCATION_FREQUENCY),
449			},
450			cmd_tx,
451		)
452	}

454	/// Report to [`Peerset`] that a substream was opened.
455	///
456	/// Slot for the stream was "preallocated" when it was initiated (outbound) or accepted
457	/// (inbound) by the local node which is why this function doesn't allocate a slot for the peer.
458	///
459	/// Returns `true` if the substream should be kept open and `false` if the substream had been
460	/// canceled while it was opening and litep2p should close the substream.
461	pub fn report_substream_opened(
462		&mut self,
463		peer: PeerId,
464		direction: traits::Direction,
465	) -> OpenResult {
466		log::trace!(
467			target: LOG_TARGET,
468			"{}: substream opened to {peer:?}, direction {direction:?}, reserved peer {}",
469			self.protocol,
470			self.reserved_peers.contains(&peer),
471		);
472
473		let Some(state) = self.peers.get_mut(&peer) else {
474			log::warn!(target: LOG_TARGET, "{}: substream opened for unknown peer {peer:?}", self.protocol);
475			debug_assert!(false);
476			return OpenResult::Reject
477		};
478
479		match state {
480			PeerState::Opening { direction: substream_direction } => {
481				let real_direction: traits::Direction = (*substream_direction).into();
482
483				*state = PeerState::Connected { direction: *substream_direction };
484				self.connected_peers.fetch_add(1usize, Ordering::Relaxed);
485
486				return OpenResult::Accept { direction: real_direction }
487			},
488			// litep2p doesn't support the ability to cancel an opening substream so if the
489			// substream was closed while it was opening, it was marked as canceled and if the
490			// substream opens succesfully, it will be closed
491			PeerState::Canceled { direction: substream_direction } => {
492				log::trace!(
493					target: LOG_TARGET,
494					"{}: substream to {peer:?} is canceled, issue disconnection request",
495					self.protocol,
496				);
497
498				self.connected_peers.fetch_add(1usize, Ordering::Relaxed);
499				*state = PeerState::Closing { direction: *substream_direction };
500
501				return OpenResult::Reject
502			},
503			// The peer was already rejected by the `report_inbound_substream` call and this
504			// should never happen. However, this code path is exercised by our fuzzer.
505			PeerState::Disconnected => {
506				log::debug!(
507					target: LOG_TARGET,
508					"{}: substream opened for a peer that was previously rejected {peer:?}",
509					self.protocol,
510				);
511				return OpenResult::Reject
512			},
513			state => {
514				log::error!(
515					target: LOG_TARGET,
516					"{}: substream opened for a peer in invalid state {peer:?}: {state:?}",
517					self.protocol,
518				);
519
520				debug_assert!(false);
521				return OpenResult::Reject;
522			},
523		}
524	}

526	/// Report to [`Peerset`] that a substream was closed.
527	///
528	/// If the peer was not a reserved peer, the inbound/outbound slot count is adjusted to account
529	/// for the disconnected peer. After the connection is closed, the peer is chilled for a
530	/// duration of [`DEFAULT_BACKOFF`] which prevens [`Peerset`] from establishing/accepting new
531	/// connections for that time period.
532	pub fn report_substream_closed(&mut self, peer: PeerId) {
533		log::trace!(target: LOG_TARGET, "{}: substream closed to {peer:?}", self.protocol);
534
535		let Some(state) = self.peers.get_mut(&peer) else {
536			log::warn!(target: LOG_TARGET, "{}: substream closed for unknown peer {peer:?}", self.protocol);
537			debug_assert!(false);
538			return
539		};
540
541		match &state {
542			// close was initiated either by remote ([`PeerState::Connected`]) or local node
543			// ([`PeerState::Closing`]) and it was a non-reserved peer
544			PeerState::Connected { direction: Direction::Inbound(Reserved::No) } |
545			PeerState::Closing { direction: Direction::Inbound(Reserved::No) } => {
546				log::trace!(
547					target: LOG_TARGET,
548					"{}: inbound substream closed to non-reserved peer {peer:?}: {state:?}",
549					self.protocol,
550				);
551
552				decrement_or_warn!(
553					self.num_in,
554					peer,
555					self.protocol,
556					Direction::Inbound(Reserved::No)
557				);
558			},
559			// close was initiated either by remote ([`PeerState::Connected`]) or local node
560			// ([`PeerState::Closing`]) and it was a non-reserved peer
561			PeerState::Connected { direction: Direction::Outbound(Reserved::No) } |
562			PeerState::Closing { direction: Direction::Outbound(Reserved::No) } => {
563				log::trace!(
564					target: LOG_TARGET,
565					"{}: outbound substream closed to non-reserved peer {peer:?} {state:?}",
566					self.protocol,
567				);
568
569				decrement_or_warn!(
570					self.num_out,
571					peer,
572					self.protocol,
573					Direction::Outbound(Reserved::No)
574				);
575			},
576			// reserved peers don't require adjustments to slot counts
577			PeerState::Closing { .. } | PeerState::Connected { .. } => {
578				log::debug!(target: LOG_TARGET, "{}: reserved peer {peer:?} disconnected", self.protocol);
579			},
580			// The peer was already rejected by the `report_inbound_substream` call and this
581			// should never happen. However, this code path is exercised by our fuzzer.
582			PeerState::Disconnected => {
583				log::debug!(
584					target: LOG_TARGET,
585					"{}: substream closed for a peer that was previously rejected {peer:?}",
586					self.protocol,
587				);
588			},
589			state => {
590				log::warn!(target: LOG_TARGET, "{}: invalid state for disconnected peer {peer:?}: {state:?}", self.protocol);
591				debug_assert!(false);
592			},
593		}
594
595		// Rejected peers do not count towards slot allocation.
596		if !matches!(state, PeerState::Disconnected) {
597			self.connected_peers.fetch_sub(1usize, Ordering::Relaxed);
598		}
599
600		*state = PeerState::Backoff;
601		self.pending_backoffs.push(Box::pin(async move {
602			Delay::new(DEFAULT_BACKOFF).await;
603			(peer, DISCONNECT_ADJUSTMENT)
604		}));
605	}

607	/// Report to [`Peerset`] that an inbound substream was opened and that it should validate it.
608	pub fn report_inbound_substream(&mut self, peer: PeerId) -> ValidationResult {
609		log::trace!(target: LOG_TARGET, "{}: inbound substream from {peer:?}", self.protocol);
610
611		if self.peerstore_handle.is_banned(&peer) {
612			log::debug!(
613				target: LOG_TARGET,
614				"{}: rejecting banned peer {peer:?}",
615				self.protocol,
616			);
617
618			return ValidationResult::Reject;
619		}
620
621		let state = self.peers.entry(peer).or_insert(PeerState::Disconnected);
622		let is_reserved_peer = self.reserved_peers.contains(&peer);
623
624		// Check if this is a non-reserved peer and if the protocol is in reserved-only mode.
625		let should_reject = self.reserved_only && !is_reserved_peer;
626
627		match state {
628			// disconnected peers that are reserved-only peers are rejected
629			PeerState::Disconnected if should_reject => {
630				log::trace!(
631					target: LOG_TARGET,
632					"{}: rejecting non-reserved peer {peer:?} in reserved-only mode (prev state: {state:?})",
633					self.protocol,
634				);
635
636				return ValidationResult::Reject
637			},
638			// disconnected peers proceed directly to inbound slot allocation
639			PeerState::Disconnected => {},
640			// peer is backed off but if it can be accepted (either a reserved peer or inbound slot
641			// available), accept the peer and then just ignore the back-off timer when it expires
642			PeerState::Backoff => {
643				if !is_reserved_peer && self.num_in == self.max_in {
644					log::trace!(
645						target: LOG_TARGET,
646						"{}: ({peer:?}) is backed-off and cannot accept, reject inbound substream",
647						self.protocol,
648					);
649
650					return ValidationResult::Reject
651				}
652
653				// The peer remains in the `PeerState::Backoff` state until the current timer
654				// expires. Then, the peer will be in the disconnected state, subject to further
655				// rejection if the peer is not reserved by then.
656				if should_reject {
657					return ValidationResult::Reject
658				}
659			},
660
661			// `Peerset` had initiated an outbound substream but litep2p had received an inbound
662			// substream before the command to open the substream was received, meaning local and
663			// remote desired to open a connection at the same time. Since outbound substreams
664			// cannot be canceled with litep2p and the command has already been registered, accept
665			// the inbound peer since the local node had wished a connection to be opened either way
666			// but keep the direction of the substream as it was (outbound).
667			//
668			// litep2p doesn't care what `Peerset` considers the substream direction to be and since
669			// it's used for bookkeeping for substream counts, keeping the substream direction
670			// unmodified simplies the implementation a lot. The direction would otherwise be
671			// irrelevant for protocols but because `SyncingEngine` has a hack to reject excess
672			// inbound substreams, that system has to be kept working for the time being. Once that
673			// issue is fixed, this approach can be re-evaluated if need be.
674			PeerState::Opening { direction: Direction::Outbound(reserved) } => {
675				if should_reject {
676					log::trace!(
677						target: LOG_TARGET,
678						"{}: rejecting inbound substream from {peer:?} ({reserved:?}) in reserved-only mode that was marked outbound",
679						self.protocol,
680					);
681
682					*state = PeerState::Canceled { direction: Direction::Outbound(*reserved) };
683					return ValidationResult::Reject
684				}
685
686				log::trace!(
687					target: LOG_TARGET,
688					"{}: inbound substream received for {peer:?} ({reserved:?}) that was marked outbound",
689					self.protocol,
690				);
691
692				return ValidationResult::Accept;
693			},
694			PeerState::Canceled { direction } => {
695				log::trace!(
696					target: LOG_TARGET,
697					"{}: {peer:?} is canceled, rejecting substream should_reject={should_reject}",
698					self.protocol,
699				);
700
701				*state = PeerState::Canceled { direction: *direction };
702				return ValidationResult::Reject
703			},
704			state => {
705				log::warn!(
706					target: LOG_TARGET,
707					"{}: invalid state ({state:?}) for inbound substream, peer {peer:?}",
708					self.protocol
709				);
710				debug_assert!(false);
711				return ValidationResult::Reject
712			},
713		}
714
715		if is_reserved_peer {
716			log::trace!(
717				target: LOG_TARGET,
718				"{}: {peer:?} accepting peer as reserved peer",
719				self.protocol,
720			);
721
722			*state = PeerState::Opening { direction: Direction::Inbound(is_reserved_peer.into()) };
723			return ValidationResult::Accept
724		}
725
726		if self.num_in < self.max_in {
727			log::trace!(
728				target: LOG_TARGET,
729				"{}: {peer:?} accepting peer as regular peer",
730				self.protocol,
731			);
732
733			self.num_in += 1;
734
735			*state = PeerState::Opening { direction: Direction::Inbound(is_reserved_peer.into()) };
736			return ValidationResult::Accept
737		}
738
739		log::trace!(
740			target: LOG_TARGET,
741			"{}: reject {peer:?}, not a reserved peer and no free inbound slots",
742			self.protocol,
743		);
744
745		*state = PeerState::Disconnected;
746		return ValidationResult::Reject
747	}

749	/// Report to [`Peerset`] that there was an error opening a substream.
750	pub fn report_substream_open_failure(&mut self, peer: PeerId, error: NotificationError) {
751		log::trace!(
752			target: LOG_TARGET,
753			"{}: failed to open substream to {peer:?}: {error:?}",
754			self.protocol,
755		);
756
757		match self.peers.get(&peer) {
758			Some(PeerState::Opening { direction: Direction::Outbound(Reserved::No) }) => {
759				decrement_or_warn!(
760					self.num_out,
761					self.protocol,
762					peer,
763					Direction::Outbound(Reserved::No)
764				);
765			},
766			Some(PeerState::Opening { direction: Direction::Inbound(Reserved::No) }) => {
767				decrement_or_warn!(
768					self.num_in,
769					self.protocol,
770					peer,
771					Direction::Inbound(Reserved::No)
772				);
773			},
774			Some(PeerState::Canceled { direction }) => match direction {
775				Direction::Inbound(Reserved::No) => {
776					decrement_or_warn!(
777						self.num_in,
778						self.protocol,
779						peer,
780						Direction::Inbound(Reserved::No)
781					);
782				},
783				Direction::Outbound(Reserved::No) => {
784					decrement_or_warn!(
785						self.num_out,
786						self.protocol,
787						peer,
788						Direction::Outbound(Reserved::No)
789					);
790				},
791				_ => {},
792			},
793			// reserved peers do not require change in the slot counts
794			Some(PeerState::Opening { direction: Direction::Inbound(Reserved::Yes) }) |
795			Some(PeerState::Opening { direction: Direction::Outbound(Reserved::Yes) }) => {
796				log::debug!(
797					target: LOG_TARGET,
798					"{}: substream open failure for reserved peer {peer:?}",
799					self.protocol,
800				);
801			},
802			state => {
803				log::debug!(
804					target: LOG_TARGET,
805					"{}: substream open failure for a unknown state: {state:?}",
806					self.protocol,
807				);
808
809				return;
810			},
811		}
812
813		self.peers.insert(peer, PeerState::Backoff);
814		self.pending_backoffs.push(Box::pin(async move {
815			Delay::new(OPEN_FAILURE_BACKOFF).await;
816			(peer, OPEN_FAILURE_ADJUSTMENT)
817		}));
818	}

820	/// [`Peerset`] had accepted a peer but it was then rejected by the protocol.
821	pub fn report_substream_rejected(&mut self, peer: PeerId) {
822		log::trace!(target: LOG_TARGET, "{}: {peer:?} rejected by the protocol", self.protocol);
823
824		match self.peers.remove(&peer) {
825			Some(PeerState::Opening { direction }) => match direction {
826				Direction::Inbound(Reserved::Yes) | Direction::Outbound(Reserved::Yes) => {
827					log::warn!(
828						target: LOG_TARGET,
829						"{}: reserved peer {peer:?} rejected by the protocol",
830						self.protocol,
831					);
832					self.peers.insert(peer, PeerState::Disconnected);
833				},
834				Direction::Inbound(Reserved::No) => {
835					decrement_or_warn!(
836						self.num_in,
837						peer,
838						self.protocol,
839						Direction::Inbound(Reserved::No)
840					);
841					self.peers.insert(peer, PeerState::Disconnected);
842				},
843				Direction::Outbound(Reserved::No) => {
844					decrement_or_warn!(
845						self.num_out,
846						peer,
847						self.protocol,
848						Direction::Outbound(Reserved::No)
849					);
850					self.peers.insert(peer, PeerState::Disconnected);
851				},
852			},
853			Some(state @ PeerState::Canceled { .. }) => {
854				log::debug!(
855					target: LOG_TARGET,
856					"{}: substream to {peer:?} rejected by protocol but already canceled",
857					self.protocol,
858				);
859
860				self.peers.insert(peer, state);
861			},
862			Some(state) => {
863				log::debug!(
864					target: LOG_TARGET,
865					"{}: {peer:?} rejected by the protocol but not opening anymore: {state:?}",
866					self.protocol,
867				);
868
869				self.peers.insert(peer, state);
870			},
871			None => {},
872		}
873	}

875	/// Calculate how many of the connected peers were counted as normal inbound/outbound peers
876	/// which is needed to adjust slot counts when new reserved peers are added.
877	///
878	/// If the peer is not already in the [`Peerset`], it is added as a disconnected peer.
879	fn calculate_slot_adjustment<'a>(
880		&'a mut self,
881		peers: impl Iterator<Item = &'a PeerId>,
882	) -> (usize, usize) {
883		peers.fold((0, 0), |(mut inbound, mut outbound), peer| {
884			match self.peers.get_mut(peer) {
885				Some(PeerState::Disconnected | PeerState::Backoff) => {},
886				Some(
887					PeerState::Opening { ref mut direction } |
888					PeerState::Connected { ref mut direction } |
889					PeerState::Canceled { ref mut direction } |
890					PeerState::Closing { ref mut direction },
891				) => {
892					*direction = match direction {
893						Direction::Inbound(Reserved::No) => {
894							inbound += 1;
895							Direction::Inbound(Reserved::Yes)
896						},
897						Direction::Outbound(Reserved::No) => {
898							outbound += 1;
899							Direction::Outbound(Reserved::Yes)
900						},
901						ref direction => **direction,
902					};
903				},
904				None => {
905					self.peers.insert(*peer, PeerState::Disconnected);
906				},
907			}
908
909			(inbound, outbound)
910		})
911	}

913	/// Checks if the peer should be disconnected based on the current state of the [`Peerset`]
914	/// and the provided direction.
915	///
916	/// Note: The role of the peer is not checked.
917	fn should_disconnect(&self, direction: Direction) -> bool {
918		match direction {
919			Direction::Inbound(_) => self.num_in >= self.max_in,
920			Direction::Outbound(_) => self.num_out >= self.max_out,
921		}
922	}
923
924	/// Increment the slot count for given peer.
925	fn increment_slot(&mut self, direction: Direction) {
926		match direction {
927			Direction::Inbound(Reserved::No) => self.num_in += 1,
928			Direction::Outbound(Reserved::No) => self.num_out += 1,
929			_ => {},
930		}
931	}

933	/// Connect to all reserved peers.
934	///
935	/// Under the following conditions:
936	/// 1. The peer must be present to the current set of peers.
937	/// 2. The peer must be disconnected.
938	/// 3. The peer must not be banned.
939	///
940	/// All reserved peers returned are transitioned to the `PeerState::Opening` state.
941	fn connect_reserved_peers(&mut self) -> Vec<PeerId> {
942		self.reserved_peers
943			.iter()
944			.filter_map(|peer| {
945				let peer_state = self.peers.get(peer);
946				if peer_state != Some(&PeerState::Disconnected) {
947					return None;
948				}
949
950				if self.peerstore_handle.is_banned(peer) {
951					return None;
952				}
953
954				// Transition peer to the opening state.
955				self.peers.insert(
956					*peer,
957					PeerState::Opening { direction: Direction::Outbound(Reserved::Yes) },
958				);
959				Some(*peer)
960			})
961			.collect::<Vec<_>>()
962	}

995impl Stream for Peerset {
996	type Item = PeersetNotificationCommand;
997
998	fn poll_next(mut self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<Option<Self::Item>> {
999		while let Poll::Ready(Some((peer, reputation))) = self.pending_backoffs.poll_next_unpin(cx)
1000		{
1001			log::trace!(target: LOG_TARGET, "{}: backoff expired for {peer:?}", self.protocol);
1002
1003			if std::matches!(self.peers.get(&peer), None | Some(PeerState::Backoff)) {
1004				self.peers.insert(peer, PeerState::Disconnected);
1005			}
1006
1007			self.peerstore_handle.report_peer(peer, reputation);
1008		}
1009
1010		if let Poll::Ready(Some(action)) = Pin::new(&mut self.cmd_rx).poll_next(cx) {
1011			log::trace!(target: LOG_TARGET, "{}: received command {action:?}", self.protocol);
1012
1013			match action {
1014				PeersetCommand::DisconnectPeer { peer } if !self.reserved_peers.contains(&peer) =>
1015					match self.peers.remove(&peer) {
1016						Some(PeerState::Connected { direction }) => {
1017							log::trace!(
1018								target: LOG_TARGET,
1019								"{}: close connection to {peer:?}, direction {direction:?}",
1020								self.protocol,
1021							);
1022
1023							self.peers.insert(peer, PeerState::Closing { direction });
1024							return Poll::Ready(Some(PeersetNotificationCommand::close_substream(
1025								vec![peer],
1026							)));
1027						},
1028						Some(PeerState::Backoff) => {
1029							log::trace!(
1030								target: LOG_TARGET,
1031								"{}: cannot disconnect {peer:?}, already backed-off",
1032								self.protocol,
1033							);
1034
1035							self.peers.insert(peer, PeerState::Backoff);
1036						},
1037						// substream might have been opening but not yet fully open when the
1038						// protocol or `Peerstore` request the connection to be closed
1039						//
1040						// if the substream opens successfully, close it immediately and mark the
1041						// peer as `Disconnected`
1042						Some(PeerState::Opening { direction }) => {
1043							log::trace!(
1044								target: LOG_TARGET,
1045								"{}: canceling substream to disconnect peer {peer:?}",
1046								self.protocol,
1047							);
1048
1049							self.peers.insert(peer, PeerState::Canceled { direction });
1050						},
1051						// protocol had issued two disconnection requests in rapid succession and
1052						// the substream hadn't closed before the second disconnection request was
1053						// received, this is harmless and can be ignored.
1054						Some(state @ PeerState::Closing { .. }) => {
1055							log::trace!(
1056								target: LOG_TARGET,
1057								"{}: cannot disconnect {peer:?}, already closing ({state:?})",
1058								self.protocol,
1059							);
1060
1061							self.peers.insert(peer, state);
1062						},
1063						// if peer is banned, e.g. due to genesis mismatch, `Peerstore` will issue a
1064						// global disconnection request to all protocols, irrespective of the
1065						// connectivity state. Peer isn't necessarily connected to all protocols at
1066						// all times so this is a harmless state to be in if a disconnection request
1067						// is received.
1068						Some(state @ PeerState::Disconnected) => {
1069							self.peers.insert(peer, state);
1070						},
1071						// peer had an opening substream earlier which was canceled and then,
1072						// e.g., the peer was banned which caused it to be disconnected again
1073						Some(state @ PeerState::Canceled { .. }) => {
1074							log::debug!(
1075								target: LOG_TARGET,
1076								"{}: cannot disconnect {peer:?}, already canceled ({state:?})",
1077								self.protocol,
1078							);
1079
1080							self.peers.insert(peer, state);
1081						},
1082						// peer doesn't exist
1083						//
1084						// this can happen, for example, when peer connects over
1085						// `/block-announces/1` and it has wrong genesis hash which initiates a ban
1086						// for that peer. Since the ban is reported to all protocols but the peer
1087						// mightn't have been registered to GRANDPA or transactions yet, the peer
1088						// doesn't exist in their `Peerset`s and the error can just be ignored.
1089						None => {
1090							log::debug!(target: LOG_TARGET, "{}: {peer:?} doesn't exist", self.protocol);
1091						},
1092					},
1093				PeersetCommand::DisconnectPeer { peer } => {
1094					log::debug!(
1095						target: LOG_TARGET,
1096						"{}: ignoring disconnection request for reserved peer {peer}",
1097						self.protocol,
1098					);
1099				},
1100				// set new reserved peers for the protocol
1101				//
1102				// Current reserved peers not in the new set are moved to the regular set of peers
1103				// or disconnected (if there are no slots available). The new reserved peers are
1104				// scheduled for outbound substreams
1105				PeersetCommand::SetReservedPeers { peers } => {
1106					log::debug!(target: LOG_TARGET, "{}: set reserved peers {peers:?}", self.protocol);
1107
1108					// reserved peers don't consume any slots so if there are any regular connected
1109					// peers, inbound/outbound slot count must be adjusted to not account for these
1110					// peers anymore
1111					//
1112					// calculate how many of the previously connected peers were counted as regular
1113					// peers and substract these counts from `num_out`/`num_in`
1114					//
1115					// If a reserved peer is not already tracked, it is added as disconnected by
1116					// `calculate_slot_adjustment`. This ensures at the next slot allocation (1sec)
1117					// that we'll try to establish a connection with the reserved peer.
1118					let (in_peers, out_peers) = self.calculate_slot_adjustment(peers.iter());
1119					self.num_out -= out_peers;
1120					self.num_in -= in_peers;
1121
1122					// collect all *reserved* peers who are not in the new reserved set
1123					let reserved_peers_maybe_remove =
1124						self.reserved_peers.difference(&peers).cloned().collect::<Vec<_>>();
1125
1126					self.reserved_peers = peers;
1127
1128					let peers_to_remove = reserved_peers_maybe_remove
1129						.into_iter()
1130						.filter(|peer| {
1131							match self.peers.remove(&peer) {
1132								Some(PeerState::Connected { mut direction }) => {
1133									// The direction contains a `Reserved::Yes` flag, because this
1134									// is a reserve peer that we want to close.
1135									// The `Reserved::Yes` ensures we don't adjust the slot count
1136									// when the substream is closed.
1137
1138									let disconnect =
1139										self.reserved_only || self.should_disconnect(direction);
1140
1141									if disconnect {
1142										log::trace!(
1143											target: LOG_TARGET,
1144											"{}: close connection to previously reserved {peer:?}, direction {direction:?}",
1145											self.protocol,
1146										);
1147
1148										self.peers.insert(*peer, PeerState::Closing { direction });
1149										true
1150									} else {
1151										log::trace!(
1152											target: LOG_TARGET,
1153											"{}: {peer:?} is no longer reserved, move to regular peers, direction {direction:?}",
1154											self.protocol,
1155										);
1156
1157										// The peer is kept connected as non-reserved. This will
1158										// further count towards the slot count.
1159										direction.set_reserved(Reserved::No);
1160										self.increment_slot(direction);
1161
1162										self.peers
1163											.insert(*peer, PeerState::Connected { direction });
1164										false
1165									}
1166								},
1167								// substream might have been opening but not yet fully open when
1168								// the protocol request the reserved set to be changed
1169								Some(PeerState::Opening { direction }) => {
1170									log::trace!(
1171										target: LOG_TARGET,
1172										"{}: cancel substream to {peer:?}, direction {direction:?}",
1173										self.protocol,
1174									);
1175
1176									self.peers.insert(*peer, PeerState::Canceled { direction });
1177									false
1178								},
1179								Some(state) => {
1180									self.peers.insert(*peer, state);
1181									false
1182								},
1183								None => {
1184									log::debug!(target: LOG_TARGET, "{}: {peer:?} doesn't exist", self.protocol);
1185									debug_assert!(false);
1186									false
1187								},
1188							}
1189						})
1190						.collect();
1191
1192					// Open substreams to the new reserved peers that are disconnected.
1193					// This ensures we are not relying on the slot allocation timer to connect to
1194					// the new reserved peers. Therefore, we start connecting to them immediately.
1195					let connect_to = self.connect_reserved_peers();
1196					let command = PeersetNotificationCommand {
1197						open_peers: connect_to,
1198						close_peers: peers_to_remove,
1199					};
1200
1201					log::trace!(
1202						target: LOG_TARGET,
1203						"{}: SetReservedPeers result {command:?}",
1204						self.protocol,
1205					);
1206
1207					return Poll::Ready(Some(command));
1208				},
1209				PeersetCommand::AddReservedPeers { peers } => {
1210					log::debug!(target: LOG_TARGET, "{}: add reserved peers {peers:?}", self.protocol);
1211
1212					// reserved peers don't consume any slots so if there are any regular connected
1213					// peers, inbound/outbound slot count must be adjusted to not account for these
1214					// peers anymore
1215					//
1216					// calculate how many of the previously connected peers were counted as regular
1217					// peers and substract these counts from `num_out`/`num_in`
1218					let (in_peers, out_peers) = self.calculate_slot_adjustment(peers.iter());
1219					self.num_out -= out_peers;
1220					self.num_in -= in_peers;
1221
1222					let peers = peers
1223						.iter()
1224						.filter_map(|peer| {
1225							if !self.reserved_peers.insert(*peer) {
1226								log::warn!(
1227									target: LOG_TARGET,
1228									"{}: {peer:?} is already a reserved peer",
1229									self.protocol,
1230								);
1231								return None
1232							}
1233
1234							std::matches!(
1235								self.peers.get_mut(peer),
1236								None | Some(PeerState::Disconnected)
1237							)
1238							.then(|| {
1239								self.peers.insert(
1240									*peer,
1241									PeerState::Opening {
1242										direction: Direction::Outbound(Reserved::Yes),
1243									},
1244								);
1245								*peer
1246							})
1247						})
1248						.collect();
1249
1250					log::debug!(target: LOG_TARGET, "{}: start connecting to {peers:?}", self.protocol);
1251
1252					return Poll::Ready(Some(PeersetNotificationCommand::open_substream(peers)));
1253				},
1254				PeersetCommand::RemoveReservedPeers { peers } => {
1255					log::debug!(target: LOG_TARGET, "{}: remove reserved peers {peers:?}", self.protocol);
1256
1257					let peers_to_remove = peers
1258						.iter()
1259						.filter_map(|peer| {
1260							if !self.reserved_peers.remove(peer) {
1261								log::debug!(
1262									target: LOG_TARGET,
1263									"{}: {peer} is not a reserved peer",
1264									self.protocol,
1265								);
1266								return None
1267							}
1268
1269							match self.peers.remove(peer)? {
1270								// peer might have already disconnected by the time request to
1271								// disconnect them was received and the peer was backed off but
1272								// it had no expired by the time the request to disconnect the
1273								// peer was received
1274								PeerState::Backoff => {
1275									log::trace!(
1276										target: LOG_TARGET,
1277										"{}: cannot disconnect removed reserved peer {peer:?}, already backed-off",
1278										self.protocol,
1279									);
1280
1281									self.peers.insert(*peer, PeerState::Backoff);
1282									None
1283								},
1284
1285								// if there is a rapid change in substream state, the peer may
1286								// be canceled when the substream is asked to be closed.
1287								//
1288								// this can happen if substream is first opened and the very
1289								// soon after canceled. The substream may not have had time to
1290								// open yet and second open is ignored. If the substream is now
1291								// closed again before it has had time to open, it will be in
1292								// canceled state since `Peerset` is still waiting to hear
1293								// either success/failure on the original substream it tried to
1294								// cancel.
1295								PeerState::Canceled { direction } => {
1296									log::trace!(
1297										target: LOG_TARGET,
1298										"{}: cannot disconnect removed reserved peer {peer:?}, already canceled",
1299										self.protocol,
1300									);
1301
1302									self.peers.insert(*peer, PeerState::Canceled { direction });
1303									None
1304								},
1305
1306								// substream to the peer might have failed to open which caused
1307								// the peer to be backed off
1308								//
1309								// the back-off might've expired by the time the peer was
1310								// disconnected at which point the peer is already disconnected
1311								// when the protocol asked the peer to be disconnected
1312								PeerState::Disconnected => {
1313									log::trace!(
1314										target: LOG_TARGET,
1315										"{}: cannot disconnect removed reserved peer {peer:?}, already disconnected",
1316										self.protocol,
1317									);
1318
1319									self.peers.insert(*peer, PeerState::Disconnected);
1320									None
1321								},
1322
1323								// if a node disconnects, it's put into `PeerState::Closing`
1324								// which indicates that `Peerset` wants the substream closed and
1325								// has asked litep2p to close it but it hasn't yet received a
1326								// confirmation. If the peer is added as a reserved peer while
1327								// the substream is closing, the peer will remain in the closing
1328								// state as `Peerset` can't do anything with the peer until it
1329								// has heard from litep2p. It's possible that the peer is then
1330								// removed from the reserved set before substream close event
1331								// has been reported to `Peerset` (which the code below is
1332								// handling) and it will once again be ignored until the close
1333								// event is heard from litep2p.
1334								PeerState::Closing { direction } => {
1335									log::trace!(
1336										target: LOG_TARGET,
1337										"{}: cannot disconnect removed reserved peer {peer:?}, already closing",
1338										self.protocol,
1339									);
1340
1341									self.peers.insert(*peer, PeerState::Closing { direction });
1342									None
1343								},
1344								// peer is currently connected as a reserved peer
1345								//
1346								// check if the peer can be accepted as a regular peer based on its
1347								// substream direction and available slots
1348								//
1349								// if there are enough slots, the peer is just converted to
1350								// a regular peer and the used slot count is increased and if the
1351								// peer cannot be accepted, litep2p is asked to close the substream.
1352								PeerState::Connected { mut direction } => {
1353									let disconnect = self.should_disconnect(direction);
1354
1355									if disconnect {
1356										log::trace!(
1357											target: LOG_TARGET,
1358											"{}: close connection to removed reserved {peer:?}, direction {direction:?}",
1359											self.protocol,
1360										);
1361
1362										self.peers.insert(*peer, PeerState::Closing { direction });
1363										Some(*peer)
1364									} else {
1365										log::trace!(
1366											target: LOG_TARGET,
1367											"{}: {peer:?} converted to regular peer {peer:?} direction {direction:?}",
1368											self.protocol,
1369										);
1370
1371										// The peer is kept connected as non-reserved. This will
1372										// further count towards the slot count.
1373										direction.set_reserved(Reserved::No);
1374										self.increment_slot(direction);
1375
1376										self.peers
1377											.insert(*peer, PeerState::Connected { direction });
1378
1379										None
1380									}
1381								},
1382
1383								PeerState::Opening { mut direction } => {
1384									let disconnect = self.should_disconnect(direction);
1385
1386									if disconnect {
1387										log::trace!(
1388											target: LOG_TARGET,
1389											"{}: cancel substream to disconnect removed reserved peer {peer:?}, direction {direction:?}",
1390											self.protocol,
1391										);
1392
1393										self.peers.insert(
1394											*peer,
1395											PeerState::Canceled {
1396												direction
1397											},
1398										);
1399									} else {
1400										log::trace!(
1401											target: LOG_TARGET,
1402											"{}: {peer:?} converted to regular peer {peer:?} direction {direction:?}",
1403											self.protocol,
1404										);
1405
1406										// The peer is kept connected as non-reserved. This will
1407										// further count towards the slot count.
1408										direction.set_reserved(Reserved::No);
1409										self.increment_slot(direction);
1410
1411										self.peers
1412											.insert(*peer, PeerState::Opening { direction });
1413									}
1414
1415									None
1416								},
1417							}
1418						})
1419						.collect();
1420
1421					log::debug!(
1422						target: LOG_TARGET,
1423						"{}: close substreams to {peers_to_remove:?}",
1424						self.protocol,
1425					);
1426
1427					return Poll::Ready(Some(PeersetNotificationCommand::close_substream(
1428						peers_to_remove,
1429					)));
1430				},
1431				PeersetCommand::SetReservedOnly { reserved_only } => {
1432					log::debug!(target: LOG_TARGET, "{}: set reserved only mode to {reserved_only}", self.protocol);
1433
1434					// update mode and if it's set to true, disconnect all non-reserved peers
1435					self.reserved_only = reserved_only;
1436
1437					if reserved_only {
1438						let peers_to_remove = self
1439							.peers
1440							.iter()
1441							.filter_map(|(peer, state)| {
1442								(!self.reserved_peers.contains(peer) &&
1443									std::matches!(state, PeerState::Connected { .. }))
1444								.then_some(*peer)
1445							})
1446							.collect::<Vec<_>>();
1447
1448						// set peers to correct states
1449
1450						// peers who are connected are move to [`PeerState::Closing`]
1451						// and peers who are already opening are moved to [`PeerState::Canceled`]
1452						// and if the substream for them opens, it will be closed right after.
1453						self.peers.iter_mut().for_each(|(_, state)| match state {
1454							PeerState::Connected { direction } => {
1455								*state = PeerState::Closing { direction: *direction };
1456							},
1457							// peer for whom a substream was opening are canceled and if the
1458							// substream opens successfully, it will be closed immediately
1459							PeerState::Opening { direction } => {
1460								*state = PeerState::Canceled { direction: *direction };
1461							},
1462							_ => {},
1463						});
1464
1465						return Poll::Ready(Some(PeersetNotificationCommand::close_substream(
1466							peers_to_remove,
1467						)));
1468					}
1469				},
1470				PeersetCommand::GetReservedPeers { tx } => {
1471					let _ = tx.send(self.reserved_peers.iter().cloned().collect());
1472				},
1473			}
1474		}
1475
1476		// periodically check if `Peerset` is currently not connected to some reserved peers
1477		// it should be connected to
1478		//
1479		// also check if there are free outbound slots and if so, fetch peers with highest
1480		// reputations from `Peerstore` and start opening substreams to these peers
1481		if let Poll::Ready(()) = Pin::new(&mut self.next_slot_allocation).poll(cx) {
1482			let mut connect_to = self.connect_reserved_peers();
1483
1484			// if the number of outbound peers is lower than the desired amount of outbound peers,
1485			// query `PeerStore` and try to get a new outbound candidated.
1486			if self.num_out < self.max_out && !self.reserved_only {
1487				// From the candidates offered by the peerstore we need to ignore:
1488				// - all peers that are not in the `PeerState::Disconnected` state (ie they are
1489				//   connected / closing)
1490				// - reserved peers since we initiated a connection to them in the previous step
1491				let ignore: HashSet<PeerId> = self
1492					.peers
1493					.iter()
1494					.filter_map(|(peer, state)| {
1495						(!std::matches!(state, PeerState::Disconnected)).then_some(*peer)
1496					})
1497					.chain(self.reserved_peers.iter().cloned())
1498					.collect();
1499
1500				let peers: Vec<_> =
1501					self.peerstore_handle.outgoing_candidates(self.max_out - self.num_out, ignore);
1502
1503				if peers.len() > 0 {
1504					peers.iter().for_each(|peer| {
1505						self.peers.insert(
1506							*peer,
1507							PeerState::Opening { direction: Direction::Outbound(Reserved::No) },
1508						);
1509					});
1510
1511					self.num_out += peers.len();
1512					connect_to.extend(peers);
1513				}
1514			}
1515
1516			// start timer for the next allocation and if there were peers which the `Peerset`
1517			// wasn't connected but should be, send command to litep2p to start opening substreams.
1518			self.next_slot_allocation = Delay::new(SLOT_ALLOCATION_FREQUENCY);
1519
1520			if !connect_to.is_empty() {
1521				log::trace!(
1522					target: LOG_TARGET,
1523					"{}: start connecting to peers {connect_to:?}",
1524					self.protocol,
1525				);
1526
1527				return Poll::Ready(Some(PeersetNotificationCommand::open_substream(connect_to)));
1528			}
1529		}
1530
1531		Poll::Pending
1532	}
