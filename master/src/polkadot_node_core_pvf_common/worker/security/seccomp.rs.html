17//! Functionality for sandboxing workers by restricting their capabilities by blocking certain
18//! syscalls with seccomp.
19//!
20//! For security we block the following:
21//!
22//! - creation of new sockets - these are unneeded in PVF jobs, and we can safely block them without
23//!   affecting consensus.
24//!
25//! - `io_uring` - allows for networking and needs to be blocked. See below for a discussion on the
26//!   safety of doing this.
27//!
28//! # Safety of blocking io_uring
29//!
30//! `io_uring` is just a way of issuing system calls in an async manner, and there is nothing
31//! stopping wasmtime from legitimately using it. Fortunately, at the moment it does not. Generally,
32//! not many applications use `io_uring` in production yet, because of the numerous kernel CVEs
33//! discovered. It's still under a lot of development. Android outright banned `io_uring` for these
34//! reasons.
35//!
36//! Considering `io_uring`'s status discussed above, and that it very likely would get detected
37//! either by our [static analysis](https://github.com/paritytech/polkadot-sdk/pull/1663) or by
38//! testing, we think it is safe to block it.
39//!
40//! ## Consensus analysis
41//!
42//! If execution hits an edge case code path unique to a given machine, it's already taken a
43//! non-deterministic branch anyway. After all, we just care that the majority of validators reach
44//! the same result and preserve consensus. So worst-case scenario, there's a dispute, and we can
45//! always admit fault and refund the wrong validator. On the other hand, if all validators take the
46//! code path that results in a seccomp violation, then they would all vote against the current
47//! candidate, which is also fine. The violation would get logged (in big scary letters) and
48//! hopefully some validator reports it to us.
49//!
50//! Actually, a worst-worse-case scenario is that 50% of validators vote against, so that there is
51//! no consensus. But so many things would have to go wrong for that to happen:
52//!
53//! 1. An update to `wasmtime` is introduced that uses io_uring (unlikely as io_uring is mainly for
54//!    IO-heavy applications)
55//!
56//! 2. The new syscall is not detected by our static analysis
57//!
58//! 3. It is never triggered in any of our tests
59//!
60//! 4. It then gets triggered on some super edge case in production on 50% of validators causing a
61//!    stall (bad but very unlikely)
62//!
63//! 5. Or, it triggers on only a few validators causing a dispute (more likely but not as bad)
64//!
65//! Considering how many things would have to go wrong here, we believe it's safe to block
66//! `io_uring`.
67//!
68//! # Action on syscall violations
69//!
70//! When a forbidden syscall is attempted we immediately kill the process in order to prevent the
71//! attacker from doing anything else. In execution, this will result in voting against the
72//! candidate.
73

78use seccompiler::*;
79use std::collections::BTreeMap;

81/// The action to take on caught syscalls.
82#[cfg(not(test))]
83const CAUGHT_ACTION: SeccompAction = SeccompAction::KillProcess;
84/// Don't kill the process when testing.
85#[cfg(test)]
86const CAUGHT_ACTION: SeccompAction = SeccompAction::Errno(libc::EACCES as u32);
87
88#[derive(thiserror::Error, Debug)]
89pub enum Error {
90	#[error(transparent)]
91	Seccomp(#[from] seccompiler::Error),
92	#[error(transparent)]
93	Backend(#[from] seccompiler::BackendError),
94	#[error("A panic occurred in try_restrict: {0}")]
95	Panic(String),
96}
97
98pub type Result<T> = std::result::Result<T, Error>;
99
100/// Try to enable seccomp for the given kind of worker.
101pub fn enable_for_worker(worker_info: &WorkerInfo) -> Result<()> {
102	gum::trace!(
103		target: LOG_TARGET,
104		?worker_info,
105		"enabling seccomp",
106	);
107
108	try_restrict()
109}
110
111/// Runs a check for seccomp in its own thread, and returns an error indicating whether seccomp with
112/// our rules is fully enabled on the current Linux environment.
113pub fn check_can_fully_enable() -> Result<()> {
114	match std::thread::spawn(|| try_restrict()).join() {
115		Ok(Ok(())) => Ok(()),
116		Ok(Err(err)) => Err(err),
117		Err(err) => Err(Error::Panic(stringify_panic_payload(err))),
118	}
119}
120
121/// Applies a `seccomp` filter to disable networking for the PVF threads.
122fn try_restrict() -> Result<()> {
123	// Build a `seccomp` filter which by default allows all syscalls except those blocked in the
124	// blacklist.
125	let mut blacklisted_rules = BTreeMap::default();
126
127	// Restrict the creation of sockets.
128	blacklisted_rules.insert(libc::SYS_socketpair, vec![]);
129	blacklisted_rules.insert(libc::SYS_socket, vec![]);
130
131	// Prevent connecting to sockets for extra safety.
132	blacklisted_rules.insert(libc::SYS_connect, vec![]);
133
134	// Restrict io_uring.
135	blacklisted_rules.insert(libc::SYS_io_uring_setup, vec![]);
136	blacklisted_rules.insert(libc::SYS_io_uring_enter, vec![]);
137	blacklisted_rules.insert(libc::SYS_io_uring_register, vec![]);
138
139	let filter = SeccompFilter::new(
140		blacklisted_rules,
141		// Mismatch action: what to do if not in rule list.
142		SeccompAction::Allow,
143		// Match action: what to do if in rule list.
144		CAUGHT_ACTION,
145		TargetArch::x86_64,
146	)?;
147
148	let bpf_prog: BpfProgram = filter.try_into()?;
149
150	// Applies filter (runs seccomp) to the calling thread.
151	seccompiler::apply_filter(&bpf_prog)?;
152
153	Ok(())
154}

156#[cfg(test)]
157mod tests {
158	use super::*;
159	use std::{io::ErrorKind, net::TcpListener, thread};
160
161	#[test]
162	fn sandboxed_thread_cannot_use_sockets() {
163		// TODO: This would be nice: <https://github.com/rust-lang/rust/issues/68007>.
164		if check_can_fully_enable().is_err() {
165			return
166		}
167
168		let handle = thread::spawn(|| {
169			// Open a socket, this should succeed before seccomp is applied.
170			TcpListener::bind("127.0.0.1:0").unwrap();
171
172			let status = try_restrict();
173			if !matches!(status, Ok(())) {
174				panic!("Ruleset should be enforced since we checked if seccomp is enabled");
175			}
176
177			// Try to open a socket after seccomp.
178			assert!(matches!(
179				TcpListener::bind("127.0.0.1:0"),
180				Err(err) if matches!(err.kind(), ErrorKind::PermissionDenied)
181			));
182
183			// Other syscalls should still work.
184			unsafe {
185				assert!(libc::getppid() > 0);
186			}
187		});
188
189		assert!(handle.join().is_ok());
190	}
191}
