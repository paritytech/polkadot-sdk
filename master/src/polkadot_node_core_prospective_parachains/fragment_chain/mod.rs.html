17//! Utility for managing parachain fragments not referenced by the relay-chain.
18//!
19//! # Overview
20//!
21//! The main type exposed by this module is the [`FragmentChain`].
22//!
23//! Each fragment chain is associated with a particular relay-parent (an active leaf) and has a
24//! [`Scope`], which contains the allowed relay parents (up to `allowed_ancestry_len`), the pending
25//! availability candidates and base constraints derived from the latest included candidate. Each
26//! parachain has a single `FragmentChain` for each active leaf where it's scheduled.
27//!
28//! A fragment chain consists mainly of the current best backable chain (we'll call this the best
29//! chain) and a storage of unconnected potential candidates (we'll call this the unconnected
30//! storage).
31//!
32//! The best chain contains all the candidates pending availability and a subsequent chain
33//! of candidates that have reached the backing quorum and are better than any other backable forks
34//! according to the fork selection rule (more on this rule later). It has a length of size at most
35//! `num_of_pending_candidates + num_of_assigned_cores_for_para`.
36//!
37//! The unconnected storage keeps a record of seconded/backable candidates that may be
38//! added to the best chain in the future.
39//!	Once a candidate is seconded, it becomes part of this unconnected storage.
40//! Only after it is backed it may be added to the best chain (but not necessarily). It's only
41//! added if it builds on the latest candidate in the chain and if there isn't a better backable
42//! candidate according to the fork selection rule.
43//!
44//! An important thing to note is that the candidates present in the unconnected storage may have
45//! any/no relationship between them. In other words, they may form N trees and may even form
46//! cycles. This is needed so that we may begin validating candidates for which we don't yet know
47//! their parent (so we may parallelize the backing process across different groups for elastic
48//! scaling) and so that we accept parachain forks.
49//!
50//! We accept parachain forks only if the fork selection rule allows for it. In other words, if we
51//! have a backed candidate, we begin seconding/validating a fork only if it has a lower candidate
52//! hash. Once both forks are backed, we discard the one with the higher candidate hash.
53//! We assume all validators pick the same fork according to the fork selection rule. If we decided
54//! to not accept parachain forks, candidates could end up getting only half of the backing votes or
55//! even less (for forks of larger arity). This would affect the validator rewards. Still, we don't
56//! guarantee that a fork-producing parachains will be able to fully use elastic scaling.
57//!
58//! Once a candidate is backed and becomes part of the best chain, we can trim from the
59//! unconnected storage candidates which constitute forks on the best chain and no longer have
60//! potential.
61//!
62//! This module also makes use of types provided by the Inclusion Emulator module, such as
63//! [`Fragment`] and [`Constraints`]. These perform the actual job of checking for validity of
64//! prospective fragments.
65//!
66//! # Fork choice rule
67//!
68//! The motivation for the fork choice rule is described in the previous chapter.
69//!
70//! The current rule is: choose the candidate with the lower candidate hash.
71//! The candidate hash is quite random and finding a candidate with a lower hash in order to favour
72//! it would essentially mean solving a proof of work problem.
73//!
74//! # Parachain cycles
75//!
76//! Parachains can create cycles, because:
77//!   1. There's no requirement that head-data is unique for a parachain. Furthermore, a parachain
78//!      is under no obligation to be acyclic, and this is mostly just because it's totally
79//!      inefficient to enforce it. Practical use-cases are acyclic, but there is still more than
80//!      one way to reach the same head-data.
81//!   2. and candidates only refer to their parent by its head-data. This whole issue could be
82//!      resolved by having candidates reference their parent by candidate hash.
83//!
84//! However, dealing with cycles increases complexity during the backing/inclusion process for no
85//! practical reason.
86//! These cycles may be accepted by fragment chains while candidates are part of the unconnected
87//! storage, but they will definitely not make it to the best chain.
88//!
89//! On the other hand, enforcing that a parachain will NEVER be acyclic would be very complicated
90//! (looping through the entire parachain's history on every new candidate or changing the candidate
91//! receipt to reference the parent's candidate hash).
92//!
93//! Therefore, we don't provide a guarantee that a cycle-producing parachain will work (although in
94//! practice they probably will if the cycle length is larger than the number of assigned cores
95//! multiplied by two).
96//!
97//! # Spam protection
98//!
99//! As long as the supplied number of candidates is bounded, [`FragmentChain`] complexity is
100//! bounded. This means that higher-level code needs to be selective about limiting the amount of
101//! candidates that are considered.
102//!
103//! Practically speaking, the collator-protocol will limit the number of fetched collations per
104//! core, to the number of claim queue assignments for the paraid on that core.
105//! Statement-distribution will not allow more than `scheduler_params.lookahead` seconded candidates
106//! at a relay parent per each validator in the backing group.
107//!
108//! The code in this module is not designed for speed or efficiency, but conceptual simplicity.
109//! Our assumption is that the amount of candidates and parachains we consider will be reasonably
110//! bounded and in practice will not exceed a few thousand at any time. This naive implementation
111//! will still perform fairly well under these conditions, despite being somewhat wasteful of
112//! memory.
113//!
114//! Still, the expensive candidate data (CandidateCommitments) are wrapped in an `Arc` and shared
115//! across fragment chains of the same para on different active leaves.

142/// Fragment chain related errors.
143#[derive(Debug, Clone, PartialEq, Error)]
144pub(crate) enum Error {
145	#[error("Candidate already known")]
146	CandidateAlreadyKnown,
147	#[error("Candidate's parent head is equal to its output head. Would introduce a cycle.")]
148	ZeroLengthCycle,
149	#[error("Candidate would introduce a cycle")]
150	Cycle,
151	#[error("Candidate would introduce two paths to the same output state")]
152	MultiplePaths,
153	#[error("Attempting to directly introduce a Backed candidate. It should first be introduced as Seconded")]
154	IntroduceBackedCandidate,
155	#[error("Relay parent {0:?} of the candidate precedes the relay parent {1:?} of a pending availability candidate")]
156	RelayParentPrecedesCandidatePendingAvailability(Hash, Hash),
157	#[error("Candidate would introduce a fork with a pending availability candidate: {0:?}")]
158	ForkWithCandidatePendingAvailability(CandidateHash),
159	#[error("Fork selection rule favours another candidate: {0:?}")]
160	ForkChoiceRule(CandidateHash),
161	#[error("Could not find parent of the candidate")]
162	ParentCandidateNotFound,
163	#[error("Could not compute candidate constraints: {0:?}")]
164	ComputeConstraints(inclusion_emulator::ModificationError),
165	#[error("Candidate violates constraints: {0:?}")]
166	CheckAgainstConstraints(inclusion_emulator::FragmentValidityError),
167	#[error("Relay parent would move backwards from the latest candidate in the chain")]
168	RelayParentMovedBackwards,
169	#[error(transparent)]
170	CandidateEntry(#[from] CandidateEntryError),
171	#[error("Relay parent {0:?} not in scope. Earliest relay parent allowed {1:?}")]
172	RelayParentNotInScope(Hash, Hash),
173}

181/// The rule for selecting between two backed candidate forks, when adding to the chain.
182/// All validators should adhere to this rule, in order to not lose out on rewards in case of
183/// forking parachains.
184fn fork_selection_rule(hash1: &CandidateHash, hash2: &CandidateHash) -> Ordering {
185	hash1.cmp(hash2)
186}

188/// Utility for storing candidates and information about them such as their relay-parents and their
189/// backing states. This does not assume any restriction on whether or not the candidates form a
190/// chain. Useful for storing all kinds of candidates.
191#[derive(Clone, Default)]
192pub(crate) struct CandidateStorage {
193	// Index from head data hash to candidate hashes with that head data as a parent. Useful for
194	// efficiency when responding to `ProspectiveValidationDataRequest`s or when trying to find a
195	// new candidate to push to a chain.
196	by_parent_head: HashMap<Hash, HashSet<CandidateHash>>,
197
198	// Index from head data hash to candidate hashes outputting that head data. For
199	// efficiency when responding to `ProspectiveValidationDataRequest`s.
200	by_output_head: HashMap<Hash, HashSet<CandidateHash>>,
201
202	// Index from candidate hash to fragment node.
203	by_candidate_hash: HashMap<CandidateHash, CandidateEntry>,
204}
205
206impl CandidateStorage {
207	/// Introduce a new pending availability candidate.
208	pub fn add_pending_availability_candidate(
209		&mut self,
210		candidate_hash: CandidateHash,
211		candidate: CommittedCandidateReceipt,
212		persisted_validation_data: PersistedValidationData,
213	) -> Result<(), Error> {
214		let entry = CandidateEntry::new(
215			candidate_hash,
216			candidate,
217			persisted_validation_data,
218			CandidateState::Backed,
219		)?;
220
221		self.add_candidate_entry(entry)
222	}
223
224	/// Return the number of stored candidates.
225	pub fn len(&self) -> usize {
226		self.by_candidate_hash.len()
227	}
228
229	/// Introduce a new candidate entry.
230	fn add_candidate_entry(&mut self, candidate: CandidateEntry) -> Result<(), Error> {
231		let candidate_hash = candidate.candidate_hash;
232		if self.by_candidate_hash.contains_key(&candidate_hash) {
233			return Err(Error::CandidateAlreadyKnown)
234		}
235
236		self.by_parent_head
237			.entry(candidate.parent_head_data_hash)
238			.or_default()
239			.insert(candidate_hash);
240		self.by_output_head
241			.entry(candidate.output_head_data_hash)
242			.or_default()
243			.insert(candidate_hash);
244		self.by_candidate_hash.insert(candidate_hash, candidate);
245
246		Ok(())
247	}
248
249	/// Remove a candidate from the store.
250	fn remove_candidate(&mut self, candidate_hash: &CandidateHash) {
251		if let Some(entry) = self.by_candidate_hash.remove(candidate_hash) {
252			if let Entry::Occupied(mut e) = self.by_parent_head.entry(entry.parent_head_data_hash) {
253				e.get_mut().remove(&candidate_hash);
254				if e.get().is_empty() {
255					e.remove();
256				}
257			}
258
259			if let Entry::Occupied(mut e) = self.by_output_head.entry(entry.output_head_data_hash) {
260				e.get_mut().remove(&candidate_hash);
261				if e.get().is_empty() {
262					e.remove();
263				}
264			}
265		}
266	}
267
268	/// Note that an existing candidate has been backed.
269	fn mark_backed(&mut self, candidate_hash: &CandidateHash) {
270		if let Some(entry) = self.by_candidate_hash.get_mut(candidate_hash) {
271			gum::trace!(target: LOG_TARGET, ?candidate_hash, "Candidate marked as backed");
272			entry.state = CandidateState::Backed;
273		} else {
274			gum::trace!(target: LOG_TARGET, ?candidate_hash, "Candidate not found while marking as backed");
275		}
276	}
277
278	/// Whether a candidate is contained within the storage already.
279	fn contains(&self, candidate_hash: &CandidateHash) -> bool {
280		self.by_candidate_hash.contains_key(candidate_hash)
281	}
282
283	/// Return an iterator over references to the stored candidates, in arbitrary order.
284	fn candidates(&self) -> impl Iterator<Item = &CandidateEntry> {
285		self.by_candidate_hash.values()
286	}
287
288	/// Try getting head-data by hash.
289	fn head_data_by_hash(&self, hash: &Hash) -> Option<&HeadData> {
290		// First, search for candidates outputting this head data and extract the head data
291		// from their commitments if they exist.
292		//
293		// Otherwise, search for candidates building upon this head data and extract the head data
294		// from their persisted validation data if they exist.
295		self.by_output_head
296			.get(hash)
297			.and_then(|m| m.iter().next())
298			.and_then(|a_candidate| self.by_candidate_hash.get(a_candidate))
299			.map(|e| &e.candidate.commitments.head_data)
300			.or_else(|| {
301				self.by_parent_head
302					.get(hash)
303					.and_then(|m| m.iter().next())
304					.and_then(|a_candidate| self.by_candidate_hash.get(a_candidate))
305					.map(|e| &e.candidate.persisted_validation_data.parent_head)
306			})
307	}
308
309	/// Returns the backed candidates which have the given head data hash as parent.
310	fn possible_backed_para_children<'a>(
311		&'a self,
312		parent_head_hash: &'a Hash,
313	) -> impl Iterator<Item = &'a CandidateEntry> + 'a {
314		let by_candidate_hash = &self.by_candidate_hash;
315		self.by_parent_head
316			.get(parent_head_hash)
317			.into_iter()
318			.flat_map(|hashes| hashes.iter())
319			.filter_map(move |h| {
320				by_candidate_hash.get(h).and_then(|candidate| {
321					(candidate.state == CandidateState::Backed).then_some(candidate)
322				})
323			})
324	}
325}

348/// Representation of a candidate into the [`CandidateStorage`].
349pub(crate) struct CandidateEntry {
350	candidate_hash: CandidateHash,
351	parent_head_data_hash: Hash,
352	output_head_data_hash: Hash,
353	relay_parent: Hash,
354	para_id: ParaId,
355	candidate: Arc<ProspectiveCandidate>,
356	state: CandidateState,
357}
358
359impl CandidateEntry {
360	/// Create a new seconded candidate entry.
361	pub fn new_seconded(
362		candidate_hash: CandidateHash,
363		candidate: CommittedCandidateReceipt,
364		persisted_validation_data: PersistedValidationData,
365	) -> Result<Self, CandidateEntryError> {
366		Self::new(candidate_hash, candidate, persisted_validation_data, CandidateState::Seconded)
367	}
368
369	pub fn hash(&self) -> CandidateHash {
370		self.candidate_hash
371	}
372
373	fn new(
374		candidate_hash: CandidateHash,
375		candidate: CommittedCandidateReceipt,
376		persisted_validation_data: PersistedValidationData,
377		state: CandidateState,
378	) -> Result<Self, CandidateEntryError> {
379		let para_id = candidate.descriptor.para_id();
380		if persisted_validation_data.hash() != candidate.descriptor.persisted_validation_data_hash()
381		{
382			return Err(CandidateEntryError::PersistedValidationDataMismatch)
383		}
384
385		let parent_head_data_hash = persisted_validation_data.parent_head.hash();
386		let output_head_data_hash = candidate.commitments.head_data.hash();
387
388		if parent_head_data_hash == output_head_data_hash {
389			return Err(CandidateEntryError::ZeroLengthCycle)
390		}
391
392		Ok(Self {
393			candidate_hash,
394			parent_head_data_hash,
395			output_head_data_hash,
396			relay_parent: candidate.descriptor.relay_parent(),
397			state,
398			candidate: Arc::new(ProspectiveCandidate {
399				commitments: candidate.commitments,
400				persisted_validation_data,
401				pov_hash: candidate.descriptor.pov_hash(),
402				validation_code_hash: candidate.descriptor.validation_code_hash(),
403			}),
404			para_id,
405		})
406	}
407}

449/// The scope of a [`FragmentChain`].

451pub(crate) struct Scope {
452	/// The relay parent we're currently building on top of.
453	relay_parent: RelayChainBlockInfo,

455	ancestors: BTreeMap<BlockNumber, RelayChainBlockInfo>,
456	/// The other relay parents candidates are allowed to build upon, mapped by the block hash.
457	ancestors_by_hash: HashMap<Hash, RelayChainBlockInfo>,
458	/// The candidates pending availability at this block.
459	pending_availability: Vec<PendingAvailability>,

463	max_backable_len: usize,

466/// An error variant indicating that ancestors provided to a scope
467/// had unexpected order.
468#[derive(Debug)]
469pub(crate) struct UnexpectedAncestor {
470	/// The block number that this error occurred at.
471	/// Allow as dead code, but it's being read in logs.
472	#[allow(dead_code)]
473	pub number: BlockNumber,
474	/// The previous seen block number, which did not match `number`.
475	/// Allow as dead code, but it's being read in logs.
476	#[allow(dead_code)]
477	pub prev: BlockNumber,
478}
479
480impl Scope {
481	/// Define a new [`Scope`].
482	///
483	/// `max_backable_len` should be the maximum length of the best backable chain (excluding
484	/// pending availability candidates).
485	///
486	/// Ancestors should be in reverse order, starting with the parent
487	/// of the `relay_parent`, and proceeding backwards in block number
488	/// increments of 1. Ancestors not following these conditions will be
489	/// rejected.
490	///
491	/// This function will only consume ancestors up to the `min_relay_parent_number` of
492	/// the `base_constraints`.
493	///
494	/// Only ancestors whose children have the same session as the relay-parent's
495	/// children should be provided.
496	///
497	/// It is allowed to provide zero ancestors.
498	pub fn with_ancestors(
499		relay_parent: RelayChainBlockInfo,
500		base_constraints: Constraints,
501		pending_availability: Vec<PendingAvailability>,
502		max_backable_len: usize,
503		ancestors: impl IntoIterator<Item = RelayChainBlockInfo>,
504	) -> Result<Self, UnexpectedAncestor> {
505		let mut ancestors_map = BTreeMap::new();
506		let mut ancestors_by_hash = HashMap::new();
507		{
508			let mut prev = relay_parent.number;
509			for ancestor in ancestors {
510				if prev == 0 {
511					return Err(UnexpectedAncestor { number: ancestor.number, prev })
512				} else if ancestor.number != prev - 1 {
513					return Err(UnexpectedAncestor { number: ancestor.number, prev })
514				} else if prev == base_constraints.min_relay_parent_number {
515					break
516				} else {
517					prev = ancestor.number;
518					ancestors_by_hash.insert(ancestor.hash, ancestor.clone());
519					ancestors_map.insert(ancestor.number, ancestor);
520				}
521			}
522		}
523
524		Ok(Scope {
525			relay_parent,
526			base_constraints,
527			max_backable_len: max_backable_len + pending_availability.len(),
528			pending_availability,
529			ancestors: ancestors_map,
530			ancestors_by_hash,
531		})
532	}
533
534	/// Get the earliest relay-parent allowed in the scope of the fragment chain.
535	pub fn earliest_relay_parent(&self) -> RelayChainBlockInfo {
536		self.ancestors
537			.iter()
538			.next()
539			.map(|(_, v)| v.clone())
540			.unwrap_or_else(|| self.relay_parent.clone())
541	}
542
543	/// Get the relay ancestor of the fragment chain by hash.
544	pub fn ancestor(&self, hash: &Hash) -> Option<RelayChainBlockInfo> {
545		if hash == &self.relay_parent.hash {
546			return Some(self.relay_parent.clone())
547		}
548
549		self.ancestors_by_hash.get(hash).map(|info| info.clone())
550	}
551
552	/// Get the base constraints of the scope
553	pub fn base_constraints(&self) -> &Constraints {
554		&self.base_constraints
555	}
556
557	/// Whether the candidate in question is one pending availability in this scope.
558	fn get_pending_availability(
559		&self,
560		candidate_hash: &CandidateHash,
561	) -> Option<&PendingAvailability> {
562		self.pending_availability.iter().find(|c| &c.candidate_hash == candidate_hash)
563	}
564}

601/// A candidate chain of backed/backable candidates.
602/// Includes the candidates pending availability and candidates which may be backed on-chain.
603#[derive(Default)]
604#[cfg_attr(test, derive(Clone))]
605struct BackedChain {
606	// Holds the candidate chain.
607	chain: Vec<FragmentNode>,
608	// Index from head data hash to the candidate hash with that head data as a parent.
609	// Only contains the candidates present in the `chain`.
610	by_parent_head: HashMap<Hash, CandidateHash>,
611	// Index from head data hash to the candidate hash outputting that head data.
612	// Only contains the candidates present in the `chain`.
613	by_output_head: HashMap<Hash, CandidateHash>,
614	// A set of the candidate hashes in the `chain`.
615	candidates: HashSet<CandidateHash>,
616}
617
618impl BackedChain {
619	fn push(&mut self, candidate: FragmentNode) {
620		self.candidates.insert(candidate.candidate_hash);
621		self.by_parent_head
622			.insert(candidate.parent_head_data_hash, candidate.candidate_hash);
623		self.by_output_head
624			.insert(candidate.output_head_data_hash, candidate.candidate_hash);
625		self.chain.push(candidate);
626	}
627
628	fn clear(&mut self) -> Vec<FragmentNode> {
629		self.by_parent_head.clear();
630		self.by_output_head.clear();
631		self.candidates.clear();
632
633		std::mem::take(&mut self.chain)
634	}
635
636	fn revert_to_parent_hash<'a>(
637		&'a mut self,
638		parent_head_data_hash: &Hash,
639	) -> impl Iterator<Item = FragmentNode> + 'a {
640		let mut found_index = None;
641		for index in 0..self.chain.len() {
642			let node = &self.chain[index];
643
644			if found_index.is_some() {
645				self.by_parent_head.remove(&node.parent_head_data_hash);
646				self.by_output_head.remove(&node.output_head_data_hash);
647				self.candidates.remove(&node.candidate_hash);
648			} else if &node.output_head_data_hash == parent_head_data_hash {
649				found_index = Some(index);
650			}
651		}
652
653		if let Some(index) = found_index {
654			self.chain.drain(min(index + 1, self.chain.len())..)
655		} else {
656			// Don't remove anything, but use drain to satisfy the compiler.
657			self.chain.drain(0..0)
658		}
659	}
660
661	fn contains(&self, hash: &CandidateHash) -> bool {
662		self.candidates.contains(hash)
663	}
664}

666/// This is the fragment chain specific to an active leaf.
667///
668/// It holds the current best backable candidate chain, as well as potential candidates
669/// which could become connected to the chain in the future or which could even overwrite the
670/// existing chain.
671#[cfg_attr(test, derive(Clone))]
672pub(crate) struct FragmentChain {
673	// The current scope, which dictates the on-chain operating constraints that all future
674	// candidates must adhere to.
675	scope: Scope,
676
677	// The current best chain of backable candidates. It only contains candidates which build on
678	// top of each other and which have reached the backing quorum. In the presence of potential
679	// forks, this chain will pick a fork according to the `fork_selection_rule`.
680	best_chain: BackedChain,
681
682	// The potential candidate storage. Contains candidates which are not yet part of the `chain`
683	// but may become in the future. These can form any tree shape as well as contain any
684	// unconnected candidates for which we don't know the parent.
685	unconnected: CandidateStorage,
686}
687
688impl FragmentChain {
689	/// Create a new [`FragmentChain`] with the given scope and populate it with the candidates
690	/// pending availability.
691	pub fn init(scope: Scope, mut candidates_pending_availability: CandidateStorage) -> Self {
692		let mut fragment_chain = Self {
693			scope,
694			best_chain: BackedChain::default(),
695			unconnected: CandidateStorage::default(),
696		};
697
698		// We only need to populate the best backable chain. Candidates pending availability must
699		// form a chain with the latest included head.
700		fragment_chain.populate_chain(&mut candidates_pending_availability);
701
702		fragment_chain
703	}
704
705	/// Populate the [`FragmentChain`] given the new candidates pending availability and the
706	/// optional previous fragment chain (of the previous relay parent).
707	pub fn populate_from_previous(&mut self, prev_fragment_chain: &FragmentChain) {
708		let mut prev_storage = prev_fragment_chain.unconnected.clone();
709
710		for candidate in prev_fragment_chain.best_chain.chain.iter() {
711			// If they used to be pending availability, don't add them. This is fine
712			// because:
713			// - if they still are pending availability, they have already been added to the new
714			//   storage.
715			// - if they were included, no point in keeping them.
716			//
717			// This cannot happen for the candidates in the unconnected storage. The pending
718			// availability candidates will always be part of the best chain.
719			if prev_fragment_chain
720				.scope
721				.get_pending_availability(&candidate.candidate_hash)
722				.is_none()
723			{
724				let _ = prev_storage.add_candidate_entry(candidate.into());
725			}
726		}
727
728		// First populate the best backable chain.
729		self.populate_chain(&mut prev_storage);
730
731		// Now that we picked the best backable chain, trim the forks generated by candidates which
732		// are not present in the best chain.
733		self.trim_uneligible_forks(&mut prev_storage, None);
734
735		// Finally, keep any candidates which haven't been trimmed but still have potential.
736		self.populate_unconnected_potential_candidates(prev_storage);
737	}

741		&self.scope
742	}

769	/// Return whether this candidate is backed in this chain or the unconnected storage.
770	pub fn is_candidate_backed(&self, hash: &CandidateHash) -> bool {
771		self.best_chain.candidates.contains(hash) ||
772			matches!(
773				self.unconnected.by_candidate_hash.get(hash),
774				Some(candidate) if candidate.state == CandidateState::Backed
775			)
776	}

778	/// Mark a candidate as backed. This can trigger a recreation of the best backable chain.
779	pub fn candidate_backed(&mut self, newly_backed_candidate: &CandidateHash) {
780		// Already backed.
781		if self.best_chain.candidates.contains(newly_backed_candidate) {
782			return
783		}
784		let Some(parent_head_hash) = self
785			.unconnected
786			.by_candidate_hash
787			.get(newly_backed_candidate)
788			.map(|entry| entry.parent_head_data_hash)
789		else {
790			// Candidate is not in unconnected storage.
791			return
792		};
793
794		// Mark the candidate hash.
795		self.unconnected.mark_backed(newly_backed_candidate);
796
797		// Revert to parent_head_hash
798		if !self.revert_to(&parent_head_hash) {
799			// If nothing was reverted, there is nothing we can do for now.
800			return
801		}
802
803		let mut prev_storage = std::mem::take(&mut self.unconnected);
804
805		// Populate the chain.
806		self.populate_chain(&mut prev_storage);
807
808		// Now that we picked the best backable chain, trim the forks generated by candidates
809		// which are not present in the best chain. We can start trimming from this candidate
810		// onwards.
811		self.trim_uneligible_forks(&mut prev_storage, Some(parent_head_hash));
812
813		// Finally, keep any candidates which haven't been trimmed but still have potential.
814		self.populate_unconnected_potential_candidates(prev_storage);
815	}

817	/// Checks if this candidate could be added in the future to this chain.
818	/// This will return `Error::CandidateAlreadyKnown` if the candidate is already in the chain or
819	/// the unconnected candidate storage.
820	pub fn can_add_candidate_as_potential(
821		&self,
822		candidate: &impl HypotheticalOrConcreteCandidate,
823	) -> Result<(), Error> {
824		let candidate_hash = candidate.candidate_hash();
825
826		if self.best_chain.contains(&candidate_hash) || self.unconnected.contains(&candidate_hash) {
827			return Err(Error::CandidateAlreadyKnown)
828		}
829
830		self.check_potential(candidate)
831	}
832
833	/// Try adding a seconded candidate, if the candidate has potential. It will never be added to
834	/// the chain directly in the seconded state, it will only be part of the unconnected storage.
835	pub fn try_adding_seconded_candidate(
836		&mut self,
837		candidate: &CandidateEntry,
838	) -> Result<(), Error> {
839		if candidate.state == CandidateState::Backed {
840			return Err(Error::IntroduceBackedCandidate);
841		}
842
843		self.can_add_candidate_as_potential(candidate)?;
844
845		// This clone is cheap, as it uses an Arc for the expensive stuff.
846		// We can't consume the candidate because other fragment chains may use it also.
847		self.unconnected.add_candidate_entry(candidate.clone())?;
848
849		Ok(())
850	}

852	/// Try getting the full head data associated with this hash.
853	pub fn get_head_data_by_hash(&self, head_data_hash: &Hash) -> Option<HeadData> {
854		// First, see if this is the head data of the latest included candidate.
855		let required_parent = &self.scope.base_constraints().required_parent;
856		if &required_parent.hash() == head_data_hash {
857			return Some(required_parent.clone())
858		}
859
860		// Cheaply check if the head data is in the best backable chain.
861		let has_head_data_in_chain = self
862			.best_chain
863			.by_parent_head
864			.get(head_data_hash)
865			.or_else(|| self.best_chain.by_output_head.get(head_data_hash))
866			.is_some();
867
868		if has_head_data_in_chain {
869			return self.best_chain.chain.iter().find_map(|candidate| {
870				if &candidate.parent_head_data_hash == head_data_hash {
871					Some(
872						candidate
873							.fragment
874							.candidate()
875							.persisted_validation_data
876							.parent_head
877							.clone(),
878					)
879				} else if &candidate.output_head_data_hash == head_data_hash {
880					Some(candidate.fragment.candidate().commitments.head_data.clone())
881				} else {
882					None
883				}
884			});
885		}
886
887		// Lastly, try getting the head data from the unconnected candidates.
888		self.unconnected.head_data_by_hash(head_data_hash).cloned()
889	}

891	/// Select `count` candidates after the given `ancestors` which can be backed on chain next.
892	///
893	/// The intention of the `ancestors` is to allow queries on the basis of
894	/// one or more candidates which were previously pending availability becoming
895	/// available or candidates timing out.
896	pub fn find_backable_chain(
897		&self,
898		ancestors: Ancestors,
899		count: u32,
900	) -> Vec<(CandidateHash, Hash)> {
901		if count == 0 {
902			return vec![]
903		}
904		let base_pos = self.find_ancestor_path(ancestors);
905
906		let actual_end_index =
907			std::cmp::min(base_pos + (count as usize), self.best_chain.chain.len());
908		let mut res = Vec::with_capacity(actual_end_index - base_pos);
909
910		for elem in &self.best_chain.chain[base_pos..actual_end_index] {
911			// Only supply candidates which are not yet pending availability. `ancestors` should
912			// have already contained them, but check just in case.
913			if self.scope.get_pending_availability(&elem.candidate_hash).is_none() {
914				res.push((elem.candidate_hash, elem.relay_parent()));
915			} else {
916				break
917			}
918		}
919
920		res
921	}
922
923	// Tries to orders the ancestors into a viable path from root to the last one.
924	// Stops when the ancestors are all used or when a node in the chain is not present in the
925	// ancestor set. Returns the index in the chain were the search stopped.
926	fn find_ancestor_path(&self, mut ancestors: Ancestors) -> usize {
927		if self.best_chain.chain.is_empty() {
928			return 0;
929		}
930
931		for (index, candidate) in self.best_chain.chain.iter().enumerate() {
932			if !ancestors.remove(&candidate.candidate_hash) {
933				return index
934			}
935		}
936
937		// This means that we found the entire chain in the ancestor set. There won't be anything
938		// left to back.
939		self.best_chain.chain.len()
940	}
941
942	// Return the earliest relay parent a new candidate can have in order to be added to the chain
943	// right now. This is the relay parent of the last candidate in the chain.
944	// The value returned may not be valid if we want to add a candidate pending availability, which
945	// may have a relay parent which is out of scope. Special handling is needed in that case.
946	// `None` is returned if the candidate's relay parent info cannot be found.
947	fn earliest_relay_parent(&self) -> Option<RelayChainBlockInfo> {
948		if let Some(last_candidate) = self.best_chain.chain.last() {
949			self.scope.ancestor(&last_candidate.relay_parent()).or_else(|| {
950				// if the relay-parent is out of scope _and_ it is in the chain,
951				// it must be a candidate pending availability.
952				self.scope
953					.get_pending_availability(&last_candidate.candidate_hash)
954					.map(|c| c.relay_parent.clone())
955			})
956		} else {
957			Some(self.scope.earliest_relay_parent())
958		}
959	}
960
961	// Return the earliest relay parent a potential candidate may have for it to ever be added to
962	// the chain. This is the relay parent of the last candidate pending availability or the
963	// earliest relay parent in scope.
964	fn earliest_relay_parent_pending_availability(&self) -> RelayChainBlockInfo {
965		self.best_chain
966			.chain
967			.iter()
968			.rev()
969			.find_map(|candidate| {
970				self.scope
971					.get_pending_availability(&candidate.candidate_hash)
972					.map(|c| c.relay_parent.clone())
973			})
974			.unwrap_or_else(|| self.scope.earliest_relay_parent())
975	}

977	// Populate the unconnected potential candidate storage starting from a previous storage.
978	fn populate_unconnected_potential_candidates(&mut self, old_storage: CandidateStorage) {
979		for candidate in old_storage.by_candidate_hash.into_values() {
980			// Sanity check, all pending availability candidates should be already present in the
981			// chain.
982			if self.scope.get_pending_availability(&candidate.candidate_hash).is_some() {
983				continue
984			}
985
986			match self.can_add_candidate_as_potential(&candidate) {
987				Ok(()) => {
988					let _ = self.unconnected.add_candidate_entry(candidate);
989				},
990				Err(e) => {
991					let msg = format!("Failed to add candidate as potential err={:?}, candidate_hash={:?}, para_id={:?}", e, candidate.candidate_hash, candidate.para_id);
992					if e.is_relay_parent_not_in_scope() {
993						gum::debug!(target: LOG_TARGET, msg);
994					} else {
995						gum::trace!(target: LOG_TARGET, msg);
996					};
997				},
998			};
999		}
1000	}

1002	// Check whether a candidate outputting this head data would introduce a cycle or multiple paths
1003	// to the same state. Trivial 0-length cycles are checked in `CandidateEntry::new`.
1004	fn check_cycles_or_invalid_tree(&self, output_head_hash: &Hash) -> Result<(), Error> {
1005		// this should catch a cycle where this candidate would point back to the parent of some
1006		// candidate in the chain.
1007		if self.best_chain.by_parent_head.contains_key(output_head_hash) {
1008			return Err(Error::Cycle)
1009		}
1010
1011		// multiple paths to the same state, which can't happen for a chain.
1012		if self.best_chain.by_output_head.contains_key(output_head_hash) {
1013			return Err(Error::MultiplePaths)
1014		}
1015
1016		Ok(())
1017	}
1018
1019	// Checks the potential of a candidate to be added to the chain now or in the future.
1020	// It works both with concrete candidates for which we have the full PVD and committed receipt,
1021	// but also does some more basic checks for incomplete candidates (before even fetching them).
1022	fn check_potential(
1023		&self,
1024		candidate: &impl HypotheticalOrConcreteCandidate,
1025	) -> Result<(), Error> {
1026		let relay_parent = candidate.relay_parent();
1027		let parent_head_hash = candidate.parent_head_data_hash();
1028
1029		// trivial 0-length cycle.
1030		if let Some(output_head_hash) = candidate.output_head_data_hash() {
1031			if parent_head_hash == output_head_hash {
1032				return Err(Error::ZeroLengthCycle)
1033			}
1034		}
1035
1036		// Check if the relay parent is in scope.
1037		let Some(relay_parent) = self.scope.ancestor(&relay_parent) else {
1038			return Err(Error::RelayParentNotInScope(
1039				relay_parent,
1040				self.scope.earliest_relay_parent().hash,
1041			))
1042		};
1043
1044		// Check if the relay parent moved backwards from the latest candidate pending availability.
1045		let earliest_rp_of_pending_availability = self.earliest_relay_parent_pending_availability();
1046		if relay_parent.number < earliest_rp_of_pending_availability.number {
1047			return Err(Error::RelayParentPrecedesCandidatePendingAvailability(
1048				relay_parent.hash,
1049				earliest_rp_of_pending_availability.hash,
1050			))
1051		}
1052
1053		// If it's a fork with a backed candidate in the current chain.
1054		if let Some(other_candidate) = self.best_chain.by_parent_head.get(&parent_head_hash) {
1055			if self.scope().get_pending_availability(other_candidate).is_some() {
1056				// Cannot accept a fork with a candidate pending availability.
1057				return Err(Error::ForkWithCandidatePendingAvailability(*other_candidate))
1058			}
1059
1060			// If the candidate is backed and in the current chain, accept only a candidate
1061			// according to the fork selection rule.
1062			if fork_selection_rule(other_candidate, &candidate.candidate_hash()) == Ordering::Less {
1063				return Err(Error::ForkChoiceRule(*other_candidate))
1064			}
1065		}
1066
1067		// Try seeing if the parent candidate is in the current chain or if it is the latest
1068		// included candidate. If so, get the constraints the candidate must satisfy.
1069		let (is_unconnected, constraints, maybe_min_relay_parent_number) =
1070			if let Some(parent_candidate) = self.best_chain.by_output_head.get(&parent_head_hash) {
1071				let Some(parent_candidate) =
1072					self.best_chain.chain.iter().find(|c| &c.candidate_hash == parent_candidate)
1073				else {
1074					// Should never really happen.
1075					return Err(Error::ParentCandidateNotFound)
1076				};
1077
1078				(
1079					false,
1080					self.scope
1081						.base_constraints
1082						.apply_modifications(&parent_candidate.cumulative_modifications)
1083						.map_err(Error::ComputeConstraints)?,
1084					self.scope.ancestor(&parent_candidate.relay_parent()).map(|rp| rp.number),
1085				)
1086			} else if self.scope.base_constraints.required_parent.hash() == parent_head_hash {
1087				// It builds on the latest included candidate.
1088				(false, self.scope.base_constraints.clone(), None)
1089			} else {
1090				// The parent is not yet part of the chain
1091				(true, self.scope.base_constraints.clone(), None)
1092			};
1093
1094		// Check for cycles or invalid tree transitions.
1095		if let Some(ref output_head_hash) = candidate.output_head_data_hash() {
1096			self.check_cycles_or_invalid_tree(output_head_hash)?;
1097		}
1098
1099		// Check against constraints if we have a full concrete candidate.
1100		if let (Some(commitments), Some(pvd), Some(validation_code_hash)) = (
1101			candidate.commitments(),
1102			candidate.persisted_validation_data(),
1103			candidate.validation_code_hash(),
1104		) {
1105			if is_unconnected {
1106				// If the parent is not yet part of the chain, we can check the commitments only
1107				// if we have the full candidate.
1108				return validate_commitments(
1109					&self.scope.base_constraints,
1110					&relay_parent,
1111					commitments,
1112					&validation_code_hash,
1113				)
1114				.map_err(Error::CheckAgainstConstraints)
1115			}
1116			Fragment::check_against_constraints(
1117				&relay_parent,
1118				&constraints,
1119				commitments,
1120				&validation_code_hash,
1121				pvd,
1122			)
1123			.map_err(Error::CheckAgainstConstraints)?;
1124		}
1125
1126		if relay_parent.number < constraints.min_relay_parent_number {
1127			return Err(Error::RelayParentMovedBackwards)
1128		}
1129
1130		if let Some(earliest_rp) = maybe_min_relay_parent_number {
1131			if relay_parent.number < earliest_rp {
1132				return Err(Error::RelayParentMovedBackwards)
1133			}
1134		}
1135
1136		Ok(())
1137	}
1138
1139	// Once the backable chain was populated, trim the forks generated by candidates which
1140	// are not present in the best chain. Fan this out into a full breadth-first search.
1141	// If `starting_point` is `Some()`, start the search from the candidates having this parent head
1142	// hash.
1143	fn trim_uneligible_forks(&self, storage: &mut CandidateStorage, starting_point: Option<Hash>) {
1144		// Start out with the candidates in the chain. They are all valid candidates.
1145		let mut queue: VecDeque<_> = if let Some(starting_point) = starting_point {
1146			[(starting_point, true)].into_iter().collect()
1147		} else {
1148			if self.best_chain.chain.is_empty() {
1149				[(self.scope.base_constraints.required_parent.hash(), true)]
1150					.into_iter()
1151					.collect()
1152			} else {
1153				self.best_chain.chain.iter().map(|c| (c.parent_head_data_hash, true)).collect()
1154			}
1155		};
1156		// To make sure that cycles don't make us loop forever, keep track of the visited parent
1157		// heads.
1158		let mut visited = HashSet::new();
1159
1160		while let Some((parent, parent_has_potential)) = queue.pop_front() {
1161			visited.insert(parent);
1162
1163			let Some(children) = storage.by_parent_head.get(&parent) else { continue };
1164			// Cannot remove while iterating so store them here temporarily.
1165			let mut to_remove = vec![];
1166
1167			for child_hash in children.iter() {
1168				let Some(child) = storage.by_candidate_hash.get(child_hash) else { continue };
1169
1170				// Already visited this parent. Either is a cycle or multiple paths that lead to the
1171				// same candidate. Either way, stop this branch to avoid looping forever.
1172				if visited.contains(&child.output_head_data_hash) {
1173					continue
1174				}
1175
1176				// Only keep a candidate if its full ancestry was already kept as potential and this
1177				// candidate itself has potential.
1178				let mut keep = false;
1179				if parent_has_potential {
1180					match self.check_potential(child) {
1181						Ok(()) => {
1182							keep = true;
1183						},
1184						Err(e) => {
1185							gum::debug!(
1186								target: LOG_TARGET,
1187								candidate_hash = ?child_hash,
1188								para_id = ?child.para_id,
1189								parent = ?parent,
1190								err = ?e,
1191								"check_potential failed for candidate"
1192							);
1193						},
1194					}
1195				}
1196
1197				if keep {
1198					queue.push_back((child.output_head_data_hash, true));
1199				} else {
1200					// Otherwise, remove this candidate and continue looping for its children, but
1201					// mark the parent's potential as `false`. We only want to remove its
1202					// children.
1203					to_remove.push(*child_hash);
1204					queue.push_back((child.output_head_data_hash, false));
1205				}
1206			}
1207
1208			for hash in to_remove {
1209				storage.remove_candidate(&hash);
1210			}
1211		}
1212	}
1213
1214	// Populate the fragment chain with candidates from the supplied `CandidateStorage`.
1215	// Can be called by the constructor or when backing a new candidate.
1216	// When this is called, it may cause the previous chain to be completely erased or it may add
1217	// more than one candidate.
1218	fn populate_chain(&mut self, storage: &mut CandidateStorage) {
1219		struct Candidate {
1220			para_id: ParaId,
1221			fragment: Fragment,
1222			candidate_hash: CandidateHash,
1223			output_head_data_hash: Hash,
1224			parent_head_data_hash: Hash,
1225		}
1226
1227		let mut cumulative_modifications =
1228			if let Some(last_candidate) = self.best_chain.chain.last() {
1229				last_candidate.cumulative_modifications.clone()
1230			} else {
1231				ConstraintModifications::identity()
1232			};
1233		let Some(mut earliest_rp) = self.earliest_relay_parent() else { return };
1234
1235		loop {
1236			if self.best_chain.chain.len() >= self.scope.max_backable_len {
1237				break;
1238			}
1239
1240			let child_constraints =
1241				match self.scope.base_constraints.apply_modifications(&cumulative_modifications) {
1242					Err(e) => {
1243						gum::debug!(
1244							target: LOG_TARGET,
1245							new_parent_head = ?cumulative_modifications.required_parent,
1246							err = ?e,
1247							"Failed to apply modifications",
1248						);
1249
1250						break
1251					},
1252					Ok(c) => c,
1253				};
1254
1255			let required_head_hash = child_constraints.required_parent.hash();
1256
1257			// Select the few possible backed/backable children which can be added to the chain
1258			// right now.
1259			let possible_children = storage
1260				.possible_backed_para_children(&required_head_hash)
1261				.filter_map(|candidate| {
1262					// Only select a candidate if:
1263					// 1. it does not introduce a fork or a cycle.
1264					// 2. parent hash is correct.
1265					// 3. relay-parent does not move backwards.
1266					// 4. all non-pending-availability candidates have relay-parent in scope.
1267					// 5. candidate outputs fulfill constraints
1268
1269					let pending = self.scope.get_pending_availability(&candidate.candidate_hash);
1270					let Some(relay_parent) = pending
1271						.map(|p| p.relay_parent.clone())
1272						.or_else(|| self.scope.ancestor(&candidate.relay_parent))
1273					else {
1274						return None
1275					};
1276
1277					if self.check_cycles_or_invalid_tree(&candidate.output_head_data_hash).is_err()
1278					{
1279						return None
1280					}
1281
1282					// require: candidates don't move backwards
1283					// and only pending availability candidates can be out-of-scope.
1284					//
1285					// earliest_rp can be before the earliest relay parent in the scope
1286					// when the parent is a pending availability candidate as well, but
1287					// only other pending candidates can have a relay parent out of scope.
1288					let min_relay_parent_number = pending
1289						.map(|p| match self.best_chain.chain.len() {
1290							0 => p.relay_parent.number,
1291							_ => earliest_rp.number,
1292						})
1293						.unwrap_or_else(|| earliest_rp.number);
1294
1295					if relay_parent.number < min_relay_parent_number {
1296						return None // relay parent moved backwards.
1297					}
1298
1299					// don't add candidates if they're already present in the chain.
1300					// this can never happen, as candidates can only be duplicated if there's a
1301					// cycle and we shouldn't have allowed for a cycle to be chained.
1302					if self.best_chain.contains(&candidate.candidate_hash) {
1303						return None
1304					}
1305
1306					let fragment = {
1307						let mut constraints = child_constraints.clone();
1308						if let Some(ref p) = pending {
1309							// overwrite for candidates pending availability as a special-case.
1310							constraints.min_relay_parent_number = p.relay_parent.number;
1311						}
1312
1313						let f = Fragment::new(
1314							relay_parent.clone(),
1315							constraints,
1316							// It's cheap to clone because it's wrapped in an Arc
1317							candidate.candidate.clone(),
1318						);
1319
1320						match f {
1321							Ok(f) => f,
1322							Err(e) => {
1323								gum::debug!(
1324									target: LOG_TARGET,
1325									err = ?e,
1326									?relay_parent,
1327									para_id = ?candidate.para_id,
1328									candidate_hash = ?candidate.candidate_hash,
1329									"Failed to instantiate fragment",
1330								);
1331
1332								return None
1333							},
1334						}
1335					};
1336
1337					let para_id = candidate.para_id;
1338
1339					Some(Candidate {
1340						para_id,
1341						fragment,
1342						candidate_hash: candidate.candidate_hash,
1343						output_head_data_hash: candidate.output_head_data_hash,
1344						parent_head_data_hash: candidate.parent_head_data_hash,
1345					})
1346				});
1347
1348			// Choose the best candidate.
1349			let best_candidate = possible_children.min_by(|lhs, rhs| {
1350				let child1 = &lhs.candidate_hash;
1351				let child2 = &rhs.candidate_hash;
1352				// Always pick a candidate pending availability as best.
1353				if self.scope.get_pending_availability(child1).is_some() {
1354					Ordering::Less
1355				} else if self.scope.get_pending_availability(child2).is_some() {
1356					Ordering::Greater
1357				} else {
1358					// Otherwise, use the fork selection rule.
1359					fork_selection_rule(child1, child2)
1360				}
1361			});
1362
1363			if let Some(Candidate {
1364				para_id,
1365				fragment,
1366				candidate_hash,
1367				output_head_data_hash,
1368				parent_head_data_hash,
1369			}) = best_candidate
1370			{
1371				// Remove the candidate from storage.
1372				storage.remove_candidate(&candidate_hash);
1373
1374				// Update the cumulative constraint modifications.
1375				cumulative_modifications.stack(fragment.constraint_modifications());
1376				// Update the earliest rp
1377				earliest_rp = fragment.relay_parent().clone();
1378
1379				let node = FragmentNode {
1380					fragment,
1381					candidate_hash,
1382					parent_head_data_hash,
1383					output_head_data_hash,
1384					cumulative_modifications: cumulative_modifications.clone(),
1385					para_id,
1386				};
1387
1388				// Add the candidate to the chain now.
1389				self.best_chain.push(node);
1390			} else {
1391				break
1392			}
1393		}
1394	}
1395
1396	// Revert the best backable chain so that the last candidate will be one outputting the given
1397	// `parent_head_hash`. If the `parent_head_hash` is exactly the required parent of the base
1398	// constraints (builds on the latest included candidate), revert the entire chain.
1399	// Return false if we couldn't find the parent head hash.
1400	fn revert_to(&mut self, parent_head_hash: &Hash) -> bool {
1401		let mut removed_items = None;
1402		if &self.scope.base_constraints.required_parent.hash() == parent_head_hash {
1403			removed_items = Some(self.best_chain.clear());
1404		}
1405
1406		if removed_items.is_none() && self.best_chain.by_output_head.contains_key(parent_head_hash)
1407		{
1408			removed_items = Some(self.best_chain.revert_to_parent_hash(parent_head_hash).collect());
1409		}
1410
1411		let Some(removed_items) = removed_items else { return false };
1412
1413		// Even if it's empty, we need to return true, because we'll be able to add a new candidate
1414		// to the chain.
1415		for node in &removed_items {
1416			let _ = self.unconnected.add_candidate_entry(node.into());
1417		}
1418		true
1419	}
1420}
