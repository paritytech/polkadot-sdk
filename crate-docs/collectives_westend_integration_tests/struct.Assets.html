pub struct Assets(/* private fields */);
Expand description

A Vec of Assets.

There are a number of invariants which the construction and mutation functions must ensure are maintained:

  • It may contain no items of duplicate asset class;
  • All items must be ordered;
  • The number of items should grow no larger than MAX_ITEMS_IN_ASSETS.

pub fn from_sorted_and_deduplicated_skip_checks(r: Vec<Asset>) -> Assets

Create a new instance of Assets from a Vec<Asset> whose contents are sorted and which contain no duplicates.

In release mode, this skips any checks to ensure that r is correct, making it a negligible-cost operation. Generally though you should avoid using it unless you have a strict proof that r is valid.

In test mode, this checks anyway and panics on fail.
